오늘 수업 목표

오늘은 C언어의 조건문을 배운다.

조건문은 "조건이 맞으면 이 코드를 실행하고, 아니면 다른 코드를 실행하라"는 문법이다.
정보처리기사 실기 C언어 문제에서 조건문은 거의 빠지지 않는다.

하지만 오늘 수업의 핵심은 단순히 if문 모양을 외우는 것이 아니다.

오늘 가장 중요한 주제는 scope이다.

scope는 우리말로 "범위"라고 한다.
C언어에서는 변수를 아무 곳에서나 쓸 수 없다.
변수를 어디에서 만들었는지에 따라 사용할 수 있는 범위가 달라진다.

특히 if문은 { } 블록을 만들기 때문에 scope를 이해하지 못하면 다음과 같은 실수를 하게 된다.

- if 안에서 만든 변수를 if 밖에서 쓰려고 한다.
- 같은 이름의 변수를 안쪽 블록에서 다시 만들어서 바깥 변수와 헷갈린다.
- if 안에서 값이 바뀐 줄 알았는데 사실 다른 변수를 바꾼 것이다.
- 조건문 안에서만 값이 대입되어, 어떤 경우에는 초기화되지 않은 값을 출력한다.

오늘 수업이 끝나면 다음을 할 수 있어야 한다.

- if문의 기본 구조를 읽을 수 있다.
- else if와 else가 언제 실행되는지 설명할 수 있다.
- 비교연산자와 논리연산자를 조건식에 사용할 수 있다.
- 이전 시간에 배운 변수, 연산자, %, ++, --를 if문과 결합할 수 있다.
- { } 블록 안과 밖의 scope 차이를 설명할 수 있다.
- if문 안에서 선언한 변수를 어디까지 사용할 수 있는지 판단할 수 있다.
- 같은 이름의 변수가 안쪽 블록에 있을 때 어떤 변수가 출력되는지 추적할 수 있다.
- 조건문이 들어간 코드를 표로 추적할 수 있다.

1. 전날 복습: 변수와 연산자
==================================================

조건문을 배우기 전에 전날 배운 내용을 짧게 복습한다.
조건문은 이전 내용과 따로 떨어진 문법이 아니다.
변수와 연산자를 이용해서 조건을 만들고, 조건에 따라 변수값을 바꾼다.

예제:

```c
#include <stdio.h>

int main() {
    int score = 75;
    int bonus = 5;

    score = score + bonus;

    printf("score = %d\n", score);

    return 0;
}
```

출력:

```text
score = 80
```

여기까지는 항상 score에 bonus를 더한다.
하지만 현실에서는 항상 더하지 않는다.

예를 들어 점수가 70점 이상일 때만 보너스를 준다면?
이때 조건문이 필요하다.

```c
if (score >= 70) {
    score = score + bonus;
}
```

이 코드는 "score가 70 이상이면 score에 bonus를 더하라"는 뜻이다.

==================================================
2. if문의 기본 구조
==================================================

if문의 기본 모양은 다음과 같다.

```c
if (조건식) {
    조건식이 참일 때 실행할 코드
}
```

읽는 방법:

```text
만약 조건식이 참이면,
중괄호 안의 코드를 실행한다.
참이 아니면,
중괄호 안의 코드를 건너뛴다.
```

예제 1:

```c
#include <stdio.h>

int main() {
    int score = 80;

    if (score >= 60) {
        printf("pass\n");
    }

    printf("end\n");

    return 0;
}
```

score는 80이다.
80 >= 60은 참이다.
따라서 if 안의 printf가 실행된다.

출력:

```text
pass
end
```

score를 50으로 바꾸면?

```c
int score = 50;
```

50 >= 60은 거짓이다.
따라서 if 안의 printf는 건너뛴다.

출력:

```text
end
```

중요:
if문은 조건이 거짓이어도 프로그램이 멈추는 것이 아니다.
if 블록만 건너뛰고 다음 줄로 내려간다.

==================================================
3. 참과 거짓
==================================================

C언어에서 조건식은 참 또는 거짓으로 판단된다.

비전공자는 처음에 "조건식"이라는 말이 어렵게 느껴질 수 있다.
조건식은 쉽게 말해 "맞다/아니다를 판단할 수 있는 식"이다.

예:

```c
score >= 60
age < 20
number % 2 == 0
a == b
```

각각은 다음 질문처럼 읽을 수 있다.

- score가 60 이상인가?
- age가 20보다 작은가?
- number를 2로 나눈 나머지가 0인가?
- a와 b가 같은가?

질문에 대한 답이 맞으면 참이다.
틀리면 거짓이다.

==================================================
4. 비교연산자
==================================================

조건식에서 가장 많이 쓰는 것은 비교연산자이다.

| 연산자 | 뜻 | 예 |
|---|---|---|
| == | 같다 | a == b |
| != | 같지 않다 | a != b |
| > | 크다 | a > b |
| >= | 크거나 같다 | a >= b |
| < | 작다 | a < b |
| <= | 작거나 같다 | a <= b |

주의:

```c
if (a = 10) {
}
```

이 코드는 "a가 10과 같은가?"가 아니다.
= 는 대입이다.

같은지 비교할 때는 반드시 ==를 사용한다.

```c
if (a == 10) {
}
```

처음에는 이 차이가 아주 중요하다.

- =  : 오른쪽 값을 왼쪽 변수에 넣는다.
- == : 왼쪽과 오른쪽이 같은지 비교한다.

5. 실습 1: 점수 합격 판정
==================================================

아래 코드를 먼저 눈으로 읽고 출력 결과를 예상하라.

```c
#include <stdio.h>

int main() {
    int score = 75;

    if (score >= 60) {
        printf("합격\n");
    }

    printf("점수: %d\n", score);

    return 0;
}
```

예상 출력:

```text
합격
점수: 75
```

변형 실습:

1. score를 60으로 바꾸면?
2. score를 59로 바꾸면?
3. 조건을 score > 60으로 바꾸면 score가 60일 때 어떻게 되는가?

정리:

- score >= 60은 60도 포함한다.
- score > 60은 60을 포함하지 않는다.

시험 문제에서는 >와 >= 차이 때문에 답이 달라지는 경우가 많다.

6. else: 아니면 실행하기
==================================================

if만 쓰면 조건이 참일 때만 무언가를 할 수 있다.
조건이 거짓일 때 다른 코드를 실행하고 싶으면 else를 사용한다.

기본 구조:

```c
if (조건식) {
    조건식이 참일 때 실행
} else {
    조건식이 거짓일 때 실행
}
```

예제 2:

```c
#include <stdio.h>

int main() {
    int score = 55;

    if (score >= 60) {
        printf("합격\n");
    } else {
        printf("불합격\n");
    }

    return 0;
}
```

score는 55이다.
55 >= 60은 거짓이다.
따라서 else 블록이 실행된다.

출력:

```text
불합격
```

if와 else는 둘 중 하나만 실행된다.
둘 다 실행되는 일은 없다.

==================================================
7. else if: 여러 조건 중 하나 고르기
==================================================

조건이 두 가지보다 많을 때는 else if를 사용한다.

예를 들어 점수에 따라 등급을 나누어 보자.

- 90 이상: A
- 80 이상: B
- 70 이상: C
- 그 외: D

코드:

```c
#include <stdio.h>

int main() {
    int score = 85;

    if (score >= 90) {
        printf("A\n");
    } else if (score >= 80) {
        printf("B\n");
    } else if (score >= 70) {
        printf("C\n");
    } else {
        printf("D\n");
    }

    return 0;
}
```

출력:

```text
B
```

왜 B인가?

위에서 아래로 검사한다.

| 검사 순서 | 조건 | 결과 |
|---|---|---|
| 1 | score >= 90 | 85 >= 90, 거짓 |
| 2 | score >= 80 | 85 >= 80, 참 |
| 3 | score >= 70 | 검사하지 않음 |
| 4 | else | 실행하지 않음 |

else if 구조에서는 한 블록이 실행되면 나머지는 검사하지 않는다.
이 점이 매우 중요하다.

==================================================
8. else if 순서가 중요한 이유
==================================================

아래 코드는 겉보기에는 비슷하지만 잘못된 코드이다.

```c
#include <stdio.h>

int main() {
    int score = 95;

    if (score >= 70) {
        printf("C\n");
    } else if (score >= 80) {
        printf("B\n");
    } else if (score >= 90) {
        printf("A\n");
    } else {
        printf("D\n");
    }

    return 0;
}
```

score는 95이다.
사람 생각으로는 A가 나와야 할 것 같다.
하지만 실제 출력은 C이다.

이유:

첫 번째 조건 score >= 70이 이미 참이기 때문이다.
if 블록이 실행되면 뒤의 else if는 더 이상 검사하지 않는다.

출력:

```text
C
```

정리:

범위를 나눌 때는 보통 큰 조건부터 검사한다.

```c
if (score >= 90) {
} else if (score >= 80) {
} else if (score >= 70) {
}
```

9. 이전 시간 내용과 결합: 나머지로 짝수/홀수 판정
==================================================

전날 배운 % 연산자를 조건문과 결합해보자.

어떤 수를 2로 나누었을 때 나머지가 0이면 짝수이다.

```c
number % 2 == 0
```

예제 3:

```c
#include <stdio.h>

int main() {
    int number = 17;

    if (number % 2 == 0) {
        printf("짝수\n");
    } else {
        printf("홀수\n");
    }

    return 0;
}
```

number는 17이다.
17 % 2는 1이다.
1 == 0은 거짓이다.
따라서 else가 실행된다.

출력:

```text
홀수
```

변수값 추적:

| 항목 | 값 |
|---|---:|
| number | 17 |
| number % 2 | 1 |
| number % 2 == 0 | 거짓 |
| 실행 블록 | else |

변형 실습:

1. number를 20으로 바꾸면?
2. number를 0으로 바꾸면?
3. 조건을 number % 2 != 0으로 바꾸면 if와 else의 출력 문구는 어떻게 바꾸는 것이 자연스러운가?

10. 이전 시간 내용과 결합: 보너스 점수 계산
==================================================

조건문은 변수값을 바꿀 때 자주 사용한다.

예제 4:

```c
#include <stdio.h>

int main() {
    int score = 78;
    int bonus = 5;

    if (score >= 70) {
        score = score + bonus;
    }

    printf("score = %d\n", score);

    return 0;
}
```

추적:

| 실행 코드 | score | bonus |
|---|---:|---:|
| int score = 78; | 78 |  |
| int bonus = 5; | 78 | 5 |
| if (score >= 70) | 참 | 5 |
| score = score + bonus; | 83 | 5 |
| printf | 83 | 5 |

출력:

```text
score = 83
```

score가 65라면?

| 실행 코드 | score | bonus |
|---|---:|---:|
| int score = 65; | 65 |  |
| int bonus = 5; | 65 | 5 |
| if (score >= 70) | 거짓 | 5 |
| score = score + bonus; | 실행 안 함 | 5 |
| printf | 65 | 5 |

출력:

```text
score = 65
```

이처럼 if문 안의 코드는 조건이 참일 때만 변수값을 바꾼다.

==================================================
11. 논리연산자: &&, ||, !
==================================================

조건이 하나가 아니라 여러 개일 때 논리연산자를 사용한다.

| 연산자 | 뜻 | 예 |
|---|---|---|
| && | 그리고 | a >= 0 && a <= 100 |
| || | 또는 | a == 0 || a == 1 |
| ! | 아니다 | !(a == 0) |

--------------------------------------------------
&&: 둘 다 참이어야 참
--------------------------------------------------

```c
if (score >= 0 && score <= 100) {
    printf("정상 점수\n");
}
```

score가 0 이상이고 100 이하일 때만 참이다.

--------------------------------------------------
||: 둘 중 하나만 참이어도 참
--------------------------------------------------

```c
if (grade == 'A' || grade == 'B') {
    printf("우수\n");
}
```

grade가 A이거나 B이면 참이다.

--------------------------------------------------
!: 참과 거짓을 뒤집음
--------------------------------------------------

```c
if (!(score >= 60)) {
    printf("불합격\n");
}
```

score >= 60이 참이면 ! 때문에 거짓이 된다.
score >= 60이 거짓이면 ! 때문에 참이 된다.

처음에는 !를 많이 쓰기보다,
읽기 쉬운 조건을 직접 쓰는 것이 좋다.

```c
if (score < 60) {
    printf("불합격\n");
}
```

12. 실습 2: 점수 범위 검사
==================================================

아래 코드는 점수가 정상 범위인지 검사한다.

```c
#include <stdio.h>

int main() {
    int score = 105;

    if (score >= 0 && score <= 100) {
        printf("정상 점수\n");
    } else {
        printf("잘못된 점수\n");
    }

    return 0;
}
```

score는 105이다.

| 조건 | 결과 |
|---|---|
| score >= 0 | 참 |
| score <= 100 | 거짓 |
| 참 && 거짓 | 거짓 |

출력:

```text
잘못된 점수
```

변형 실습:

1. score = -1이면?
2. score = 0이면?
3. score = 100이면?
4. &&를 ||로 바꾸면 왜 문제가 생기는가?

주의:

```c
if (score >= 0 || score <= 100)
```

이 조건은 거의 모든 정수에서 참이 된다.
예를 들어 105는 score >= 0이 참이라서 전체가 참이다.
-1은 score <= 100이 참이라서 전체가 참이다.

점수가 0 이상 100 이하인지 보려면 반드시 &&를 써야 한다.

==================================================
13. 중괄호 { }와 블록
==================================================

이제 오늘의 가장 중요한 주제인 scope로 들어간다.
scope를 이해하려면 먼저 블록을 알아야 한다.

C언어에서 { }로 감싼 부분을 블록이라고 한다.

```c
if (score >= 60) {
    printf("pass\n");
    printf("good\n");
}
```

위 코드에서 { 부터 } 까지가 하나의 블록이다.

블록은 "이 코드들이 한 묶음이다"라는 뜻이다.
if 조건이 참이면 블록 전체가 실행된다.
if 조건이 거짓이면 블록 전체가 건너뛰어진다.

--------------------------------------------------
중괄호가 없으면 한 줄만 if에 포함된다
--------------------------------------------------

아래 코드를 보자.

```c
#include <stdio.h>

int main() {
    int score = 50;

    if (score >= 60)
        printf("합격\n");
        printf("수고했습니다\n");

    return 0;
}
```

들여쓰기만 보면 printf 두 줄이 모두 if 안에 있는 것처럼 보인다.
하지만 C언어는 들여쓰기로 범위를 판단하지 않는다.
중괄호가 없으면 if 바로 다음 한 문장만 if에 포함된다.

실제 의미:

```c
if (score >= 60) {
    printf("합격\n");
}
printf("수고했습니다\n");
```

score는 50이므로 "합격"은 출력되지 않는다.
하지만 "수고했습니다"는 if 밖에 있으므로 항상 출력된다.

출력:

```text
수고했습니다
```

처음 배울 때는 if문에 항상 중괄호를 쓰는 습관을 들이는 것이 좋다.

==================================================
14. scope란 무엇인가
==================================================

scope는 변수를 사용할 수 있는 범위이다.

변수는 선언된 위치에 따라 사용할 수 있는 곳이 정해진다.

가장 중요한 규칙:

```text
변수는 자신이 선언된 블록 안에서만 사용할 수 있다.
안쪽 블록에서는 바깥 블록의 변수를 사용할 수 있다.
바깥 블록에서는 안쪽 블록의 변수를 사용할 수 없다.
```

예를 들어 main 함수도 하나의 블록이다.

```c
int main() {
    int a = 10;

    if (a > 0) {
        int b = 20;
    }

    return 0;
}
```

여기서 a는 main 블록 안에서 선언되었다.
따라서 main 블록 안에서는 사용할 수 있다.

b는 if 블록 안에서 선언되었다.
따라서 if 블록 안에서만 사용할 수 있다.
if 블록이 끝난 뒤에는 b를 사용할 수 없다.

15. scope 예제 1: 바깥 변수는 안쪽에서 쓸 수 있다
==================================================

```c
#include <stdio.h>

int main() {
    int score = 80;

    if (score >= 60) {
        printf("score = %d\n", score);
    }

    return 0;
}
```

score는 main 블록에서 선언되었다.
if 블록은 main 블록 안쪽에 있다.
안쪽 블록에서는 바깥 블록의 변수를 사용할 수 있다.

따라서 이 코드는 정상이다.

출력:

```text
score = 80
```

그림으로 보면 다음과 같다.

```text
main 블록
{
    score 선언

    if 블록
    {
        score 사용 가능
    }
}
```

16. scope 예제 2: 안쪽 변수는 바깥에서 쓸 수 없다
==================================================

아래 코드는 컴파일 오류가 난다.

```c
#include <stdio.h>

int main() {
    int score = 80;

    if (score >= 60) {
        int bonus = 5;
        printf("bonus = %d\n", bonus);
    }

    printf("bonus = %d\n", bonus);

    return 0;
}
```

왜 오류인가?

bonus는 if 블록 안에서 선언되었다.
따라서 if 블록 안에서만 사용할 수 있다.
if 블록이 끝난 뒤에는 bonus라는 변수를 사용할 수 없다.

문제되는 줄:

```c
printf("bonus = %d\n", bonus);
```

이 줄은 if 블록 밖에 있다.
그래서 bonus를 찾을 수 없다.

흔히 나오는 오류 메시지:

```text
'bonus' undeclared
```

또는

```text
bonus was not declared in this scope
```

이 말은 "bonus라는 이름을 이 범위에서 찾을 수 없다"는 뜻이다.

해결 방법:
if 밖에서도 bonus를 쓰고 싶다면, bonus를 if 밖에서 선언해야 한다.

```c
#include <stdio.h>

int main() {
    int score = 80;
    int bonus = 0;

    if (score >= 60) {
        bonus = 5;
    }

    printf("bonus = %d\n", bonus);

    return 0;
}
```

출력:

```text
bonus = 5
```

17. scope 예제 3: if 안에서 값 바꾸기
==================================================

아래 코드는 정상이다.

```c
#include <stdio.h>

int main() {
    int score = 80;
    int bonus = 0;

    if (score >= 60) {
        bonus = 5;
        score = score + bonus;
    }

    printf("score = %d\n", score);
    printf("bonus = %d\n", bonus);

    return 0;
}
```

score와 bonus는 둘 다 main 블록에서 선언되었다.
if 블록은 main 안쪽에 있으므로 score와 bonus를 사용할 수 있다.
그리고 if 안에서 값을 바꾸면 main 블록의 변수값이 실제로 바뀐다.

추적:

| 실행 코드 | score | bonus |
|---|---:|---:|
| int score = 80; | 80 |  |
| int bonus = 0; | 80 | 0 |
| if (score >= 60) | 참 | 0 |
| bonus = 5; | 80 | 5 |
| score = score + bonus; | 85 | 5 |
| printf | 85 | 5 |

출력:

```text
score = 85
bonus = 5
```

18. scope 예제 4: 같은 이름의 변수
==================================================

이제 조금 어려운 예제를 보자.

```c
#include <stdio.h>

int main() {
    int a = 10;

    if (a > 0) {
        int a = 20;
        printf("if 안: %d\n", a);
    }

    printf("if 밖: %d\n", a);

    return 0;
}
```

출력:

```text
if 안: 20
if 밖: 10
```

왜 이런 일이 생길까?

main 블록에서 a를 만들었다.

```c
int a = 10;
```

그런데 if 블록 안에서 다시 같은 이름의 a를 만들었다.

```c
int a = 20;
```

이 경우 if 블록 안에서는 안쪽 a가 바깥 a를 가린다.
이것을 shadowing이라고 부른다.
오늘은 영어 이름을 외우는 것보다 "안쪽 변수가 바깥 변수를 가린다" 정도로 이해하면 된다.

그림:

```text
main 블록의 a = 10
{
    if 블록
    {
        if 블록의 a = 20
        여기서 a를 출력하면 20
    }

    if 블록이 끝나면 안쪽 a는 사라짐
    여기서 a를 출력하면 main 블록의 a = 10
}
```

중요:

```c
int a = 20;
```

이 코드는 기존 a의 값을 20으로 바꾸는 것이 아니다.
새로운 a를 하나 더 만드는 것이다.

기존 a를 바꾸고 싶다면 int를 다시 쓰면 안 된다.

```c
a = 20;
```

19. scope 예제 5: int를 붙이면 새 변수, 안 붙이면 기존 변수
==================================================

두 코드를 비교하라.

--------------------------------------------------
코드 A
--------------------------------------------------

```c
#include <stdio.h>

int main() {
    int a = 10;

    if (a > 0) {
        int a = 20;
    }

    printf("%d\n", a);

}
```

출력:

```text

```

if 안의 int a = 20;은 새로운 변수를 만든 것이다.
바깥 a는 바뀌지 않는다.

--------------------------------------------------
코드 B
--------------------------------------------------

```c
#include <stdio.h>

}
```

```text
20
```

if 안의 a = 20;은 기존 a에 20을 대입한 것이다.

```c
int a = 20;  // 새 변수 선언
```

20. scope 예제 6: 조건에 따라 초기화되지 않을 수 있다
==================================================

아래 코드를 보자.

```c
#include <stdio.h>

int main() {
    int score = 50;
    int result;

    if (score >= 60) {
        result = 1;
    }

    printf("%d\n", result);

    return 0;
}
```

이 코드는 위험하다.

score는 50이다.
score >= 60은 거짓이다.
따라서 if 안의 result = 1;은 실행되지 않는다.

그런데 result는 처음에 값을 넣지 않았다.

```c
int result;
```

값이 정해지지 않은 변수를 출력하는 것은 잘못된 코드이다.
컴파일러나 실행 환경에 따라 이상한 값이 나올 수 있다.

좋은 코드:

```c
#include <stdio.h>

int main() {
    int score = 50;
    int result = 0;

    if (score >= 60) {
        result = 1;
    }

    printf("%d\n", result);

    return 0;
}
```

출력:

```text
0
```

또는 else를 사용한다.

```c
#include <stdio.h>

int main() {
    int score = 50;
    int result;

    if (score >= 60) {
        result = 1;
    } else {
        result = 0;
    }

    printf("%d\n", result);

    return 0;
}
```

이 코드는 score가 어떤 값이든 result에 반드시 값이 들어간다.

21. 조건문과 scope를 함께 추적하는 방법
==================================================

조건문이 들어간 코드는 다음 순서로 추적한다.

1. 변수의 초기값을 표에 쓴다.
2. if 조건식을 계산한다.
3. 참이면 if 블록으로 들어간다.
4. 거짓이면 if 블록을 건너뛴다.
5. 블록 안에서 새로 선언된 변수인지, 기존 변수에 대입한 것인지 확인한다.
6. 블록이 끝나면 안쪽에서 선언한 변수는 사라진다고 생각한다.
7. 마지막 출력 위치에서 어떤 변수를 볼 수 있는지 확인한다.

추적표에는 다음 칸을 넣으면 좋다.

| 줄 | 조건 결과 | 바깥 변수 | 안쪽 변수 | 출력 |
|---|---|---|---|---|

특히 같은 이름의 변수가 있을 때는 "바깥 a", "if 안 a"처럼 나누어 적는다.

22. 실습 3: 같은 이름 변수 추적

```c

        int x = 100;

        printf("A: %d\n", x);

    x += 2;

}
```

| 위치 | 바깥 x | if 안 x | 설명 |

| printf B | 7 | 없음 | B는 7 출력 |

```text

B: 7
```

이 문제의 핵심:

```c
int x = 100;
```

if 블록 안에서만 사용할 수 있는 새로운 x를 만든다.

24. 실습 5: if와 else의 scope
==================================================

if 블록과 else 블록은 서로 다른 블록이다.

```c
#include <stdio.h>

int main() {
    int score = 40;

    if (score >= 60) {
        int message = 1;
        printf("if message = %d\n", message);
    } else {
        int message = 2;
        printf("else message = %d\n", message);
    }

    return 0;
}
```

출력:

```text
else message = 2
```

if 블록의 message와 else 블록의 message는 서로 다른 변수이다.
둘 다 각자의 블록 안에서만 사용할 수 있다.

다음 코드는 오류이다.

```c
#include <stdio.h>

int main() {
    int score = 40;

    if (score >= 60) {
        int message = 1;
    } else {
        int message = 2;
    }

    printf("%d\n", message);

    return 0;
}
```

이유:
message는 if 블록 또는 else 블록 안에서만 선언되었다.
블록 밖에서는 message를 사용할 수 없다.

해결:

```c
#include <stdio.h>

int main() {
    int score = 40;
    int message = 0;

    if (score >= 60) {
        message = 1;
    } else {
        message = 2;
    }

    printf("%d\n", message);

    return 0;
}
```

출력:

```text
2
```

29. 흔한 실수 2: == 대신 =
==================================================

아래 코드는 처음 배우는 사람이 자주 실수한다.

```c
#include <stdio.h>

int main() {
    int a = 5;

    if (a = 10) {
        printf("A\n");
    }

    printf("a = %d\n", a);

    return 0;
}
```

이 코드는 a가 10인지 비교하는 코드가 아니다.

```c
a = 10
```

이것은 a에 10을 대입하는 코드이다.
C언어에서는 0이 아닌 값은 참처럼 판단된다.
10은 0이 아니므로 if문은 참으로 처리된다.

출력:

```text
A
a = 10
```

45. 오늘의 핵심 정리
==================================================

1. if는 조건이 참일 때만 블록을 실행한다.

```c
if (조건식) {
}
```

2. else는 if 조건이 거짓일 때 실행한다.

```c
if (조건식) {
} else {
}
```

3. else if는 여러 조건 중 위에서부터 처음 참인 하나만 실행한다.

```c
if (조건1) {
} else if (조건2) {
} else {
}
```

4. 비교할 때는 ==를 사용한다.

```c
a == 10
```

5. 대입할 때는 =를 사용한다.

```c
a = 10;
```

6. 조건식에는 전날 배운 연산자를 같이 쓴다.

```c
n % 2 == 0
score + bonus >= 80
a / b > 3
```

7. scope는 변수를 사용할 수 있는 범위이다.

```text
변수는 자신이 선언된 블록 안에서만 사용할 수 있다.
```

8. 바깥 변수는 안쪽 블록에서 사용할 수 있다.

```c
int a = 10;
if (a > 0) {
    printf("%d\n", a);
}
```

9. 안쪽 블록에서 선언한 변수는 바깥에서 사용할 수 없다.

```c
if (a > 0) {
    int b = 20;
}
printf("%d\n", b);  // 오류
```

10. int를 다시 붙이면 새 변수 선언이다.

```c
int a = 10;
if (a > 0) {
    int a = 20;  // 새 변수
}
printf("%d\n", a);  // 10
```

11. int 없이 대입하면 기존 변수 수정이다.

```c
int a = 10;
if (a > 0) {
    a = 20;  // 기존 변수 수정
}
printf("%d\n", a);  // 20
```

12. if 여러 개와 else if는 다르다.

- if 여러 개: 각각 독립적으로 검사
- if / else if: 처음 참인 하나만 실행

따라서 if 블록이 실행된다.

```c
a = a / b;
```

14 / 5는 정수 나눗셈이므로 2이다.

출력:

```text
2 5
```

--------------------------------------------------
숙제 2 정답
--------------------------------------------------

출력:

```text
1
```

score는 92이다.
score >= 60이 첫 번째 조건에서 이미 참이다.
따라서 grade = 1이 실행되고 아래 else if는 검사하지 않는다.

3이 나오지 않는 이유:
else if는 위에서부터 처음 참인 하나만 실행하기 때문이다.

올바른 순서:

```c
if (score >= 90) {
    grade = 3;
} else if (score >= 80) {
    grade = 2;
} else if (score >= 60) {
    grade = 1;
}
```

--------------------------------------------------
숙제 3 정답
--------------------------------------------------

오류가 난다.

result는 if 블록 또는 else 블록 안에서 각각 선언되었다.
블록 밖의 printf에서는 result를 사용할 수 없다.

수정:

```c
#include <stdio.h>

int main() {
    int n = 8;
    int result = 0;

    if (n % 2 == 0) {
        result = n / 2;
    } else {
        result = n * 2;
    }

    printf("%d\n", result);

    return 0;
}
```

n은 8이고 8 % 2 == 0은 참이다.
result = 8 / 2 = 4이다.

출력:

```text
4
```

--------------------------------------------------
숙제 4 정답
--------------------------------------------------

초기:

| 변수 | 값 |
|---|---:|
| x | 3 |
| 바깥 y | 4 |

조건:
3 < 4는 참이다.

if 안:

```c
int y = 10;
```

if 안 y가 새로 만들어진다.

```c
x = x + y;
```

여기서 y는 if 안 y이다.
x = 3 + 10 = 13이다.

```c
y++;
```

if 안 y가 11이 된다.

첫 번째 출력:

```text
13 11
```

if가 끝나면 if 안 y는 사라진다.
바깥 y는 여전히 4이다.
x는 13으로 바뀌어 있다.

두 번째 출력:

```text
13 4
```

최종 출력:

```text
13 11
13 4
```

--------------------------------------------------
숙제 5 정답
--------------------------------------------------

초기:

| a | b | result |
|---:|---:|---:|
| 9 | 2 | 0 |

첫 번째 if:

```c
if (a % 2 == 1)
```

9 % 2는 1이다.
1 == 1은 참이다.

if 블록 실행:

```c
result = a / b;
```

9 / 2는 정수 나눗셈이므로 4이다.

```c
a--;
```

a는 8이 된다.

첫 번째 if 이후:

| a | b | result |
|---:|---:|---:|
| 8 | 2 | 4 |

두 번째 if:

```c
if (result >= 4)
```

4 >= 4는 참이다.

```c
result += a;
```

result = 4 + 8 = 12이다.

최종:

| a | b | result |
|---:|---:|---:|
| 8 | 2 | 12 |

출력:

```text
8 2 12
```

==================================================
마무리 멘트
==================================================

오늘 배운 조건문은 앞으로 반복문, 배열, 함수 문제를 풀 때 계속 나온다.
하지만 조건문보다 더 중요한 것은 scope이다.

코드를 볼 때마다 다음 질문을 해야 한다.

1. 이 변수는 어디에서 선언되었는가?
2. 지금 이 위치에서 이 변수를 사용할 수 있는가?
3. if 안에서 int를 다시 썼는가, 아니면 기존 변수에 대입했는가?
4. if 여러 개인가, else if인가?
5. 조건이 참일 때와 거짓일 때 각각 어떤 값이 바뀌는가?

이 질문을 습관처럼 하면 C언어 출력 결과 문제에서 실수가 크게 줄어든다.

다음 수업에서는 반복문 for, while을 배운다.
반복문은 오늘 배운 조건식과 scope를 그대로 사용한다.
그래서 오늘의 if와 scope 이해가 다음 수업의 출발점이다.
