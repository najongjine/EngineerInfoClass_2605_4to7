2. 구조체는 왜 필요할까?

2-1. 구조체가 없을 때: 값은 있지만 관계가 보이지 않는다

학생 한 명을 저장하려고 합니다. 이름은 문자열, 점수는 정수, 등급은 문자입니다. 각각 변수로 만들 수는 있지만, 학생이 늘어나면 같은 학생의 값끼리 연결해 관리하는 일이 급격히 어려워집니다.

흩어진 변수 방식 - 변수 이름의 규칙으로만 "김민지의 정보"임을 억지로 표시합니다.

char name1[]김민지
int score187
char grade1B
char name2[]박지훈
int score294
char grade2A
char name1[20] = "Minji";
int score1 = 87;
char grade1 = 'B';

char name2[20] = "Jihun";
int score2 = 94;
char grade2 = 'A';

이 방식은 학생 한 명의 값을 함수로 보내거나 배열로 정렬할 때 특히 불편합니다. 이름 배열과 점수 배열의 인덱스가 어긋나면 "이름은 민지인데 점수는 지훈"인 오류가 생깁니다.

2-2. 구조체가 있을 때: 한 대상을 한 묶음으로 표현한다

구조체 방식 - 서로 다른 자료형의 값을 Student라는 한 기록(record)으로 묶습니다.

이름Minji
점수87
등급B
→
struct Student minji
nameMinji
score87
gradeB
배열과의 차이

배열은 보통 같은 자료형의 여러 값입니다. 예: int scores[3].

구조체의 역할

구조체는 한 대상의 서로 다른 성격의 값을 묶습니다. 예: 이름 + 점수 + 등급.

한 줄 정의: 구조체(struct)는 관련 있는 여러 값을 하나의 사용자 정의 자료형으로 묶어, 현실의 한 대상처럼 다룰 수 있게 하는 C언어 문법입니다.

3. 구조체 선언과 멤버 접근

3-1. 틀을 만들고, 실제 변수를 만든다

struct Student는 아직 학생 한 명의 값이 아니라, "학생 정보는 이렇게 생겼다"라는 설계도(자료형)입니다. 설계도 뒤에 s1을 선언해야 실제 저장 공간이 생깁니다.

#include <stdio.h>
#include <string.h>

struct Student {
    char name[20];
    int score;
    char grade;
};

int main(void) {
    struct Student s1;

    strcpy(s1.name, "Minji");
    s1.score = 87;
    s1.grade = 'B';

    printf("%s %d %c\n", s1.name, s1.score, s1.grade);
    return 0;
}
코드읽는 방법결과
struct Student { ... };Student 형태를 정의한다틀만 만들어진다
struct Student s1;Student형 변수 s1을 만든다실제 저장 칸이 생긴다
s1.score = 87;s1의 score 멤버에 87 저장점 연산자 . 사용
strcpy(s1.name, "Minji");s1의 문자 배열에 문자열 복사문자열 배열은 통째 대입 불가
초보자가 자주 틀리는 부분: s1.name = "Minji";는 배열 멤버에 나중에 문자열을 대입하려는 코드이므로 허용되지 않습니다. 선언과 동시에 초기화하거나, 이미 만든 뒤에는 strcpy를 사용합니다.

3-2. 선언과 동시에 초기화하기

struct Student s1 = {"Minji", 87, 'B'};
struct Student s2 = {"Jihun", 94, 'A'};

printf("%d\n", s1.score + s2.score);  // 181

중괄호의 값은 구조체에 적힌 멤버 순서대로 들어갑니다. 이름 → 점수 → 등급 순서를 바꾸면 의도한 자료가 되지 않습니다.

3-3. typedef로 자료형 이름을 짧게 만들기

시험 코드에서는 typedef가 함께 나오는 일이 많습니다. 복잡한 새 기능이 아니라, 긴 이름의 별명을 만드는 문법으로 읽으면 됩니다.

typedef struct {
    char name[20];
    int score;
    char grade;
} Student;

int main(void) {
    Student a = {"Ara", 80, 'B'};
    printf("%s:%d\n", a.name, a.score);
    return 0;
}

struct Student s;

구조체 태그 이름 앞에 매번 struct를 적는 방식입니다.

Student s;

typedef로 Student를 새 자료형 이름처럼 쓰는 방식입니다.

3-4. 점(.)과 화살표(->)를 구별한다

구조체 변수 자체를 들고 있으면 점을 씁니다. 구조체의 주소를 저장한 포인터가 있으면 화살표를 씁니다. 지난 시간의 포인터가 오늘 구조체와 연결되는 지점입니다.

s.score
상자 자체로 접근
Student s
|
Student *p = &s
p->score
주소로 접근

같은 칸을 가리키는 표현: p->score는 (*p).score와 같습니다. 괄호가 필요한 이유는 .의 계산 우선순위가 *보다 높기 때문입니다.

Student s = {"Yuna", 70, 'C'};
Student *p = &s;

p->score += 15;
p->grade = 'B';

printf("%d %c\n", s.score, s.grade);  // 85 B

4. 구조체 배열: 여러 대상을 같은 규칙으로 관리한다

학생 한 명은 구조체 변수 하나로 충분합니다. 학생 세 명을 같은 방식으로 저장하려면 구조체를 원소로 갖는 배열을 만듭니다. 배열이므로 [i]로 한 학생을 고르고, 점 연산자로 그 학생의 멤버를 고릅니다.

Student list[3]의 모습

list[0]
Ara78C
list[1]
Bora91A
list[2]
Chan84B

list[1]은 두 번째 학생 전체, list[1].score는 그 학생 안의 점수 91입니다.

typedef struct {
    char name[20];
    int score;
    char grade;
} Student;

int main(void) {
    Student list[3] = {
        {"Ara", 78, 'C'},
        {"Bora", 91, 'A'},
        {"Chan", 84, 'B'}
    };
    int i, sum = 0;

    for (i = 0; i < 3; i++) {
        sum += list[i].score;
        printf("%s:%d\n", list[i].name, list[i].score);
    }
    printf("avg=%d\n", sum / 3);
    return 0;
}
반복ilist[i].score계산 전 sum계산 후 sum
1회차078078
2회차19178169
3회차284169253

정수끼리 나누므로 평균 출력은 253 / 3의 정수 몫인 84입니다. 구조체 문제 안에서도 기존에 배운 정수 나눗셈 규칙은 그대로 적용됩니다.

최고점 학생 찾기의 흐름

int top = 0;
for (int i = 1; i < 3; i++) {
    if (list[i].score > list[top].score) {
        top = i;
    }
}
printf("%s\n", list[top].name);

top에는 최고 점수 자체가 아니라 최고 학생의 위치(인덱스)가 저장됩니다. 이름도 점수도 같은 학생에게서 꺼내기 위해 위치를 기억하는 것입니다.

5. 구조체와 "자료 결합도"를 정확히 이해하기

여기서 용어가 헷갈리기 쉽습니다. 구조체는 여러 자료를 한 대상의 기록으로 묶는 C언어의 자료형 기능입니다. 반면 결합도(coupling)는 모듈이나 함수가 서로 얼마나 강하게 의존하는지를 설명하는 설계 개념입니다.

중요한 용어 교정: 구조체 전체를 함수에 넘겼다고 해서 곧바로 "자료 결합도(data coupling)"라고 부르지는 않습니다. 필요한 단순 값만 전달하면 자료 결합도, 여러 멤버를 가진 구조체처럼 자료구조 전체를 전달하면 보통 스탬프 결합도(stamp coupling, 자료구조 결합도)라고 구분합니다.

5-1. 같은 학생 자료, 다른 함수 전달 방식

자료 결합도 · Data Coupling 
char getGrade(int score) {
    return score >= 90 ? 'A' : 'B';
}

grade = getGrade(s.score);

함수가 판단에 필요한 score 하나만 받습니다. 함수 입장에서는 학생의 이름이나 학번을 알 필요가 없습니다.

스탬프 결합도 · Stamp Coupling 
char getGrade(Student s) {
    return s.score >= 90 ? 'A' : 'B';
}

grade = getGrade(s);

판단에는 점수만 필요하지만 학생 구조체 전체를 받습니다. 함수가 자료구조의 모양에 의존합니다.

6. 코딩 실습: 읽고, 추적하고, 직접 바꾸기

아래 실습은 컴파일 전에 먼저 출력 결과를 손으로 예상합니다. 그 다음 실제 실행 결과와 비교하고, 마지막으로 지시된 값을 바꾸어 새 문제로 만듭니다.

실습 A. 구조체 한 개 만들고 출력하기

#include <stdio.h>

typedef struct {

int main(void) {

실습 C. 구조체 포인터로 원본 값 수정하기

함수의 매개변수가 구조체 자체면 복사본을 고치고, 구조체 포인터면 원본 칸을 고칠 수 있습니다. 이 차이는 출력 추적 문제의 단골입니다.

#include <stdio.h>

typedef struct {
    char name[10];
    int score;
} Student;

void bonus(Student *p, int point) {
    p->score += point;
    if (p->score > 100) {
        p->score = 100;
    }
}

int main(void) {
    Student s = {"Kim", 88};
    bonus(&s, 15);
    printf("%s %d\n", s.name, s.score);
    return 0;
}
main의 s
nameKim
score88 → 100
←
p = &s
p->score로 수정

88 + 15는 103이지만, 함수 내부의 조건문 때문에 다시 100이 됩니다. 출력은 Kim 100입니다.

실습 D. 값 전달과 주소 전달 비교

#include <stdio.h>

typedef struct {
    int score;
} Student;

void changeA(Student s) {
    s.score += 10;
}

void changeB(Student *s) {
    s->score += 10;
}

int main(void) {
    Student x = {70};
    changeA(x);
    printf("%d ", x.score);
    changeB(&x);
    printf("%d\n", x.score);
    return 0;
}
호출함수 안에서 수정하는 대상호출 후 x.score
changeA(x)x의 복사본70 그대로

void addBonus(Student *s, int bonus)를 작성하세요. 점수에 보너스를 더하되 결과가 100을 넘으면 100으로 만듭니다.

확인 데이터: {"Lee", 96}에 7점을 주면 100이 출력되어야 합니다.

문제 8-9 예시 답안 보기 
typedef struct {
    char name[20];
    int score;
} Student;

void addBonus(Student *s, int bonus) {
    s->score += bonus;
    if (s->score > 100)
        s->score = 100;
}

void printAboveAverage(Student list[], int size) {
    int i, total = 0, avg;
    for (i = 0; i < size; i++)
        total += list[i].score;

    avg = total / size;
    for (i = 0; i < size; i++) {
        if (list[i].score >= avg)
            printf("%s ", list[i].name);
    }
}

배열을 함수로 넘기면 원소에 접근할 수 있고, Student *s는 학생 한 명의 원본을 수정할 때 사용합니다.

8. 혼자 공부하는 학생을 위한 복습 루트

오늘 반드시 이해할 다섯 문장

  • 구조체는 한 대상의 관련 있는 여러 값을 묶는 사용자 정의 자료형이다.
  • .는 구조체 변수의 멤버를, ->는 구조체 포인터가 가리킨 대상의 멤버를 선택한다.
  • list[i].score는 i번째 구조체 원소 안의 점수 멤버이다.
  • 필요 값만 함수에 넘기는 것은 자료 결합도, 구조체 전체를 넘기는 것은 스탬프 결합도로 구분한다.
  • 실기 코드에서는 이전 줄에서 바뀐 멤버값으로 다음 줄을 계산해야 한다.

오답 방지 체크리스트

  • 구조체 설계도 선언 끝의 세미콜론 ;을 썼다.
  • 구조체 변수에는 ., 포인터에는 ->를 사용했다.
  • 문자 배열 멤버를 선언 이후 바꿀 때 strcpy를 사용했다.
  • 구조체 배열에서 list[i].member 순서를 혼동하지 않았다.
  • 함수가 구조체 복사본을 받는지, 주소를 받아 원본을 바꾸는지 확인했다.
  • 구조체 전체 전달을 무조건 자료 결합도라고 쓰지 않고 스탬프 결합도와 구분했다.
