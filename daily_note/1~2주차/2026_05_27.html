1. 함수는 일을 나누는 작은 기계다

함수는 어떤 값을 받아서, 정해진 일을 한 뒤, 결과를 돌려주는 코드 묶음이다. C언어에서 함수 문제를 풀 때는 함수 이름보다 호출 순서, 매개변수 값, return 값을 먼저 봐야 한다.

함수 호출 흐름 시각화

main 시작
add(3, 4) 호출
a=3, b=4 복사
return 7
main으로 복귀
#include <stdio.h>

int add(int a, int b) {
    int result = a + b;
    return result;
}

int main() {
    int x = 3;
    int y = 4;
    int z = add(x, y);

    printf("%d\n", z);
    return 0;
}

손으로 추적하기

순서 실행 위치 x y a b result z

매개변수 a, b는 x, y와 이름이 다르다. 함수가 호출될 때 값이 복사되어 들어간다. 그래서 일반 변수 전달에서는 함수 안에서 매개변수를 바꿔도 main의 원본 변수는 그대로다.

값 전달: 복사본을 바꾸면 원본은 안 바뀐다

#include <stdio.h>

void change(int n) {
    n = 100;
    printf("change 안 n: %d\n", n);
}

int main() {
    int x = 10;

    change(x);
    printf("main 안 x: %d\n", x);

    return 0;
}
출력 순서 출력 이유
1 change 안 n: 100 x의 값 10이 n으로 복사된 뒤, n만 100으로 바뀐다.
2 main 안 x: 10 n은 함수가 끝나면 사라지고, 원본 x는 그대로다.

2. 지역변수와 전역변수

변수는 아무 데서나 보이는 것이 아니다. 변수가 만들어진 위치에 따라 볼 수 있는 범위가 달라진다. 이 범위를 scope라고 한다.

지역변수

함수나 블록 안에서 만든 변수다. 그 안에서만 사용하고, 블록이 끝나면 사라진다.

전역변수

함수 밖에서 만든 변수다. 여러 함수에서 접근할 수 있지만, 시험 문제에서는 값 변화 추적이 까다로워진다.

#include <stdio.h>

int score = 50;

void test() {
    int score = 80;
    printf("%d\n", score);
}

int main() {
    test();
    printf("%d\n", score);
    return 0;
}

같은 이름이어도 다른 칸일 수 있다

전역 공간
score50
test 함수 내부
score80

지역변수가 전역변수 이름을 가린다.

출력은 80, 50이다. test 안에서는 가까운 지역변수 score를 먼저 본다. main에서 출력할 때는 test의 지역변수가 이미 사라졌으므로 전역변수 score를 본다.

3. 포인터는 주소를 저장하는 변수다

변수는 값을 저장하는 칸이다. 그런데 C언어에서는 그 칸이 메모리의 어디에 있는지도 다룰 수 있다. 주소는 변수 칸의 위치이고, 포인터는 그 주소를 저장하는 변수다.

&x

x라는 변수 칸의 주소를 뜻한다.

int *p

정수형 변수의 주소를 저장할 수 있는 포인터 변수 p를 만든다.

*p

p가 가리키는 주소로 가서, 그 칸의 값을 읽거나 바꾼다.

#include <stdio.h>

int main() {
    int x = 10;
    int *p = &x;

    printf("%d\n", x);
    printf("%d\n", *p);

    *p = 30;
    printf("%d\n", x);

    return 0;
}

포인터 그림: p는 x의 주소를 들고 있다

포인터 변수 p 0x100
0x100 10
0x104 ?
0x108 ?
0x10C ?
x
다른 칸
다른 칸
다른 칸

*p = 30은 p를 바꾸는 것이 아니다

초보자가 가장 많이 헷갈리는 부분이다. p = &x는 포인터 변수 p 안에 주소를 넣는 것이다. *p = 30은 p가 들고 있는 주소로 이동해서, 그곳의 값을 30으로 바꾸는 것이다.

코드 뜻 바뀌는 것
p = &x; p가 x를 가리키게 한다. p 안의 주소값
*p = 30; p가 가리키는 칸에 30을 넣는다. x의 값
printf("%d", *p); p가 가리키는 칸의 값을 출력한다. 출력만 함

포인터로 함수 밖 변수 바꾸기

#include <stdio.h>

void change(int *p) {
    *p = 100;
}

int main() {
    int x = 10;

    change(&x);
    printf("%d\n", x);

    return 0;
}

주소 전달 흐름

main의 x = 10
change(&x)
p가 x 주소 저장
*p = 100
main의 x가 100
값 전달은 복사본을 바꾸는 방식이고, 주소 전달은 원본 칸을 찾아가서 바꾸는 방식이다. 실기 문제에서 함수가 변수 값을 실제로 바꾸는지 판단하려면 매개변수에 *가 있는지, 호출할 때 &를 넘기는지 확인한다.

4. 배열과 포인터는 왜 비슷해 보일까?

C언어 시험에서 포인터와 배열은 자주 같이 나온다. 이유는 배열의 이름이 많은 상황에서 첫 번째 칸의 주소처럼 사용되기 때문이다. 

#include <stdio.h>

int main() {
    int arr[4] = {10, 20, 30, 40};
    int *p = arr;

    printf("%d\n", arr[0]);
    printf("%d\n", *arr);
    printf("%d\n", p[0]);
    printf("%d\n", *p);

    printf("%d\n", arr[2]);
    printf("%d\n", *(arr + 2));
    printf("%d\n", p[2]);
    printf("%d\n", *(p + 2));

    return 0;
}

배열 메모리 그림

0x200 10
0x204 20
0x208 30
0x20C 40
arr[0]
arr[1]
arr[2]
arr[3]
arr 또는 p 0x200

arr은 첫 칸 주소처럼 쓰이고, p도 첫 칸 주소를 저장한다.

arr == &arr[0]
*arr == arr[0]
arr + 2 == &arr[2]
*(arr + 2) == arr[2]
p[i] == *(p + i)
arr[i] == *(arr + i)
시험 풀이 관점에서는 arr와 &arr[0]를 거의 같은 출발 주소로 보고 추적하면 된다. 다만 C언어 문법상 배열 변수와 포인터 변수는 완전히 같은 물건은 아니다. int *p는 다른 주소를 다시 넣을 수 있지만, arr이라는 배열 이름 자체에 다른 배열 주소를 대입할 수는 없다.

포인터 덧셈은 주소가 1씩 증가한다는 뜻이 아니다

int 배열에서 arr + 1은 주소 숫자에 1을 더하는 것이 아니라, 다음 int 칸으로 이동한다는 뜻이다. 보통 int가 4바이트라면 주소는 0x200에서 0x204로 이동한다.

표현 가리키는 칸 값
arr, arr + 0, &arr[0] 첫 번째 칸 10
arr + 1, &arr[1] 두 번째 칸 20
arr + 2, &arr[2] 세 번째 칸 30

배열을 함수에 넘기면 포인터처럼 받는다

#include <stdio.h>

void changeFirst(int arr[]) {
    arr[0] = 99;
}

int main() {
    int nums[3] = {1, 2, 3};

    changeFirst(nums);
    printf("%d\n", nums[0]);

    return 0;
}

출력은 99이다. 배열을 함수에 넘길 때 배열 전체가 복사되는 것이 아니라 첫 칸 주소가 넘어간다고 생각하면 된다. 그래서 함수 안의 arr[0] 수정은 main의 nums[0] 수정으로 이어진다.

5. 코딩 실습

아래 실습은 반드시 "예상 -> 손 추적 -> 실행 -> printf 로그 추가 -> 값 바꾸기" 순서로 진행한다. 코드를 바로 복사해서 실행하기 전에 결과를 먼저 적어야 한다.

실습 1: 함수 return 흐름

#include <stdio.h>

int calc(int a, int b) {
    a = a + 2;
    b = b * 3;
    return a + b;
}

int main() {
    int x = 1;
    int y = 2;
    int result = calc(x, y);

    printf("x=%d y=%d result=%d\n", x, y, result);
    return 0;
}
확인할 것 질문
값 전달 calc 안에서 a, b를 바꾸면 x, y도 바뀌는가?
return result에는 어떤 값이 들어가는가?

실습 2: 포인터로 두 변수 바꾸기

#include <stdio.h>

void swapLike(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 7;
    int y = 4;

    swapLike(&x, &y);
    printf("x=%d y=%d\n", x, y);

    return 0;
}

swapLike 추적 그림

a는 x 주소
b는 y 주소
temp = *a = 7
*a = *b -> x=4
*b = temp -> y=7

실습 3: 배열과 포인터 인덱싱

#include <stdio.h>

int main() {
    int arr[5] = {2, 4, 6, 8, 10};
    int *p = arr;

    printf("%d\n", *(p + 1));
    printf("%d\n", p[3]);

    *(p + 2) = 60;
    p[4] = p[0] + p[1];

    for (int i = 0; i < 5; i++) {
        printf("%d ", arr[i]);
    }

    return 0;
}
단계 arr[0] arr[1] arr[2] arr[3] arr[4]
처음 2 4 6 8 10
*(p + 2) = 60 후 2 4 60 8 10
p[4] = p[0] + p[1] 후 2 4 60 8 6

실습 4: 로그를 추가해서 확인하기

다음 코드를 실행하기 전에, 함수 안에 로그를 직접 추가한다.

#include <stdio.h>

void addIndex(int *arr, int size) {
    for (int i = 0; i < size; i++) {
        arr[i] = arr[i] + i;
    }
}

int main() {
    int data[4] = {5, 5, 5, 5};

    addIndex(data, 4);

    for (int i = 0; i < 4; i++) {
        printf("%d ", data[i]);
    }

    return 0;
}
추가할 로그 예시: printf("i=%d arr[i]=%d\n", i, arr[i]); 이 로그를 대입 전과 대입 후에 각각 넣으면 값이 바뀌는 순간을 볼 수 있다.

6. 실전 실기수준 연습문제

실제 시험처럼 출력 결과를 먼저 적는다. 답을 바로 열지 말고, 각 문제마다 변수표 또는 메모리표를 만든 뒤 확인한다.

함수와 지역변수

#include <stdio.h>

int f(int x) {
    x = x + 5;
    return x * 2;
}

int main() {
    int x = 3;
    int y = f(x);

    printf("%d %d\n", x, y);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 3 16

f(x)를 호출할 때 x의 값 3이 함수 매개변수 x로 복사된다. 함수 안 x는 8이 되고, return x * 2로 16을 돌려준다. main의 x는 복사본이 아니므로 그대로 3이다.

전역변수와 지역변수

#include <stdio.h>

int n = 10;

void test() {
    int n = 3;
    n++;
    printf("%d ", n);
}

int main() {
    test();
    printf("%d\n", n);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 4 10

test 안의 int n = 3은 지역변수다. n++ 후 4가 출력된다. main에서 출력하는 n은 전역변수이므로 10이다.

포인터로 값 변경

#include <stdio.h>

void update(int *p) {
    *p = *p + 7;
}

int main() {
    int a = 5;
    update(&a);
    printf("%d\n", a);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 12

update(&a)에서 a의 주소가 전달된다. *p는 a 자체를 의미하므로 a = a + 7과 같은 효과가 난다.

값 전달과 주소 전달 비교

#include <stdio.h>

void one(int x) {
    x = x + 10;
}

void two(int *x) {
    *x = *x + 10;
}

int main() {
    int a = 1;

    one(a);
    two(&a);

    printf("%d\n", a);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 11

one(a)는 값 복사라서 main의 a가 바뀌지 않는다. two(&a)는 주소 전달이라서 원본 a가 1에서 11로 바뀐다.

배열 이름과 포인터

#include <stdio.h>

int main() {
    int arr[4] = {3, 6, 9, 12};
    int *p = arr;

    printf("%d ", *p);
    printf("%d ", *(p + 2));
    printf("%d\n", p[3]);

    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 3 9 12

p는 arr[0]의 주소를 가진다. *p는 arr[0], *(p + 2)는 arr[2], p[3]은 arr[3]과 같다.

배열을 함수에서 수정

#include <stdio.h>

void work(int *p) {
    p[0] = p[1] + p[2];
    *(p + 2) = p[0] - p[1];
}

int main() {
    int data[3] = {2, 4, 6};

    work(data);
    printf("%d %d %d\n", data[0], data[1], data[2]);

    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 10 4 6

처음에는 {2, 4, 6}이다. p[0] = p[1] + p[2]로 첫 번째 값이 10이 된다. 그다음 *(p + 2)는 p[2]와 같으므로 p[2] = 10 - 4가 되어 6이다.

반복문과 포인터

#include <stdio.h>

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    int *p = arr;
    int sum = 0;

    for (int i = 0; i < 5; i += 2) {
        sum += *(p + i);
    }

    printf("%d\n", sum);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 9

i는 0, 2, 4로 변한다. 더하는 값은 arr[0], arr[2], arr[4]이므로 1 + 3 + 5 = 9이다.

실수 찾기

#include <stdio.h>

void change(int p) {
    *p = 20;
}

int main() {
    int x = 5;
    change(&x);
    printf("%d\n", x);
    return 0;
}

이 코드는 컴파일 오류가 난다. 잘못된 부분을 고치시오.

정답과 해설

수정: void change(int *p)

change(&x)는 x의 주소를 넘긴다. 주소를 받으려면 매개변수도 포인터인 int *p여야 한다. 그래야 *p = 20으로 원본 x를 바꿀 수 있다.

7. 오늘의 마무리 정리

함수
  • 함수 호출은 main에서 다른 함수로 갔다가 return을 만나 다시 돌아오는 흐름이다.
  • 일반 매개변수는 값 복사라서 함수 안에서 바꿔도 원본이 바뀌지 않는다.
  • 지역변수는 함수나 블록이 끝나면 사라진다.
  • 전역변수는 여러 함수에서 볼 수 있지만, 같은 이름의 지역변수가 있으면 지역변수가 우선이다.
포인터와 배열
  • &x는 x의 주소다.
  • int *p는 int 변수의 주소를 저장하는 포인터다.
  • *p는 p가 가리키는 칸의 값이다.
  • 배열 이름은 많은 상황에서 첫 칸 주소처럼 쓰인다.
  • arr[i]와 *(arr + i)는 같은 칸을 읽는다.

시험장에서 보는 순서

  1. 함수 호출이 나오면 main에서 시작점을 찾는다.
  2. 매개변수가 일반 변수인지 포인터인지 확인한다.
  3. 일반 변수면 값 복사, 포인터면 원본 수정 가능성을 생각한다.
  4. 배열이 나오면 인덱스표를 먼저 만든다.
  5. p[i], *(p+i), arr[i], *(arr+i)를 같은 방식으로 바꿔서 본다.
  6. 출력문이 여러 개면 출력 순서를 따로 적는다.
오늘의 금지 습관: 포인터 문제를 눈으로만 넘기기. 포인터는 머릿속으로만 보면 금방 흐려진다. 주소 칸, 값 칸, 화살표를 그리면 훨씬 덜 무섭다.

숙제

  • 연습문제 1-8번을 다시 풀고, 각 문제마다 변수표 또는 메모리표를 작성한다.
  • 실습 3의 배열값을 {5, 10, 15, 20, 25}로 바꾸고 최종 출력 결과를 다시 예측한다.
  • 실습 4에서 arr[i] = arr[i] + i;를 arr[i] = arr[i] * i;로 바꾸면 결과가 어떻게 달라지는지 확인한다.
  • 책에서는 함수, 지역변수, 전역변수, 포인터 기본 문법의 예제를 읽고 &, *, []가 나오는 줄에 표시한다.
