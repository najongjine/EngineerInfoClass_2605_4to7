1. 어제의 포인터를 오늘의 배열로 연결하기

포인터는 주소를 저장하는 변수다. 주소를 따라가면 그 칸의 값을 읽거나 바꿀 수 있다. 배열은 같은 자료형의 칸을 메모리에 연속으로 놓아 둔 것이다. 그러므로 배열 첫 칸 주소를 포인터에 넣으면, 포인터가 옆 칸으로 이동하며 배열 전체를 읽을 수 있다. 

#include <stdio.h>

int main() {
    int score[4] = {70, 80, 90, 100};
    int *p = score;

    printf("%d\n", *p);        // score[0]
    printf("%d\n", *(p + 2));  // score[2]

    return 0;
}

메모리는 칸, 포인터는 칸의 주소표

int *p = score 0x100

score[0]의 주소로 출발한다.

0x100 70 score[0], *p
0x104 80 score[1]
0x108 90 score[2], *(p+2)
0x10C 100 score[3]

주소는 설명용 예시다. int가 4바이트인 환경을 그렸기 때문에 다음 칸은 4씩 이동한다.

기호를 한 문장으로 읽기

표현 말로 읽기 위 그림의 결과
score 대부분의 식에서 score의 첫 번째 칸 주소 0x100
p p가 저장하고 있는 주소 0x100
*p p의 주소에 있는 값을 읽기 70
p + 2 int 두 칸 뒤의 주소 0x108
*(p + 2) int 두 칸 뒤의 값을 읽기 90

2. 배열과 포인터: 시험에서는 같은 칸으로 번역한다

배열의 인덱스 표기 []는 포인터 이동과 참조를 짧게 적은 문법으로 해석할 수 있다. 시험에서 arr[i]와 *(arr + i)가 섞여 나오면 모양만 다를 뿐 같은 배열 칸이라고 바꾸어 읽으면 된다.

arr == &arr[0]
*arr == arr[0]
arr + i == &arr[i]
*(arr + i) == arr[i]
p[i] == *(p + i)
&p[i] == p + i
#include <stdio.h>

int main() {
    int data[5] = {3, 6, 9, 12, 15};
    int *p = data;

    printf("%d %d %d\n", data[2], *(data + 2), p[2]);

    *(p + 1) = 60;
    data[4] = *(p + 1) + *p;

    for (int i = 0; i < 5; i++) {
        printf("%d ", data[i]);
    }
    return 0;
}

한 줄씩 값의 변화 추적

순서 실행한 문장 data[0] data[1] data[2] data[3] data[4] 출력
1 초기화 3 6 9 12 15
2 printf(...data[2]...) 3 6 9 12 15 9 9 9
3 *(p + 1) = 60 3 60 9 12 15
4 data[4] = *(p+1) + *p 3 60 9 12 63
5 for 출력 3 60 9 12 63 3 60 9 12 63

중요한 정정: 비슷하게 쓰이지만 완전히 같은 물건은 아니다

비교 항목 int arr[3] 배열 int *p 포인터
공통점 arr[i], p[i], *(arr+i), *(p+i)로 연결된 칸의 값을 읽을 수 있다.
정체 int 값을 넣어 둔 3칸 자체다. 어딘가의 주소 하나를 저장하는 변수다.
다른 곳을 가리키기 arr = other; 불가 p = other; 가능
sizeof 전체 배열 크기: 예를 들어 12바이트 주소 하나 크기: 환경에 따라 보통 4 또는 8바이트
정확한 결론: 문제를 읽을 때 배열 이름은 첫 칸 주소처럼 사용되므로 포인터식으로 계산한다. 그러나 배열 자체와 주소를 보관하는 포인터 변수는 선언, 대입, 크기 계산에서 서로 다르다. "접근할 때는 매우 비슷하다. 물건 자체는 다르다."라고 기억한다.

3. 문자열은 문자를 담은 배열이다

C언어에는 문자열을 따로 보관하는 기본 자료형이 없다. 문자열 "CODE"는 char형 칸에 문자 하나씩 저장한 배열이고, 끝났다는 표시로 '\0' 한 칸이 더 들어간다. 

char word[5] = {'C', 'O', 'D', 'E', '\0'};
char same[] = "CODE";  // 컴파일러가 마지막 '\0'까지 넣어 준다.

"CODE"의 실제 배열 모양

word + 0 'C' word[0]
word + 1 'O' word[1]
word + 2 'D' word[2]
word + 3 'E' word[3]
word + 4 '\0' 출력 종료 표지
char *p = word &word[0]

포인터는 'C'부터 한 칸씩 이동하여 '\0'에서 멈춘다.

문자 하나

char ch = 'A';

작은따옴표. 한 칸에 문자 하나를 넣는다.

문자열

char text[] = "A";

큰따옴표. 실제로는 'A', '\0' 두 칸이다.

왜 '\0'가 반드시 필요할까?

배열은 어디까지가 사용 중인 문자인지 스스로 알지 못한다. printf("%s", word)는 첫 칸부터 읽기 시작해 '\0'를 만날 때까지 출력한다. 끝 표시가 빠지면 배열 바깥의 쓰레기 값까지 읽으려 하므로 정상적인 문자열이 아니다. 

#include <stdio.h>

int main() {
    char word[] = "CODE";

    printf("%s\n", word);    // CODE
    printf("%c\n", word[1]); // O
    printf("%d\n", word[4]); // '\0'의 정수값 0

    return 0;
}
실기 읽기 규칙: %s가 보이면 문자 배열을 '\0'까지 읽는다. %c가 보이면 그 위치의 문자 한 칸만 출력한다. word[i]는 *(word+i)로 바꾸어도 같은 문자다.

4. 포인터로 문자열을 걷고, 함수에서 원본을 바꾼다

숫자 배열을 함수에 넘길 때와 문자 배열을 함수에 넘길 때 원리는 같다. 배열 이름은 첫 칸 주소처럼 전달되므로, 함수는 원본 문자 칸을 직접 읽고 바꿀 수 있다.

포인터로 문자열 길이 세기

#include <stdio.h>

int length(char *p) {
    int count = 0;

    while (*p != '\0') {
        count++;
        p++;
    }

    return count;
}

int main() {
    char word[] = "CODE";
    printf("%d\n", length(word));
    return 0;
}

length("CODE") 이동 흐름

*p = 'C'
count = 1
계속 이동
*p = 'O'
count = 2
계속 이동
*p = 'D'
count = 3
계속 이동
*p = 'E'
count = 4
계속 이동
*p = '\0'
조건 거짓
return 4

함수에서 소문자를 대문자로 바꾸기

영문 소문자 'a'부터 'z'는 연속된 문자 코드로 배치되어 있다. 따라서 소문자라면 'a' - 'A'만큼 빼서 대문자로 만들 수 있다. 

#include <stdio.h>

void upper(char *p) {
    while (*p != '\0') {
        if (*p >= 'a' && *p <= 'z') {
            *p = *p - ('a' - 'A');
        }
        p++;
    }
}

int main() {
    char text[] = "c-Study";

    upper(text);
    printf("%s\n", text);
    return 0;
}
i 읽은 문자 소문자인가? 바뀐 문자 문자열 상태
0 'c' 예 'C' "C-Study"
1 '-' 아니오 '-' "C-Study"
2 'S' 아니오 'S' "C-Study"
3-6 't','u','d','y' 예 'T','U','D','Y' "C-STUDY"
함수 안에서 p++를 해도 main의 배열 이름 text가 사라지거나 이동하는 것은 아니다. 함수의 포인터 변수 p가 원본 배열의 각 칸을 차례대로 가리키며, *p = ...일 때만 해당 원본 칸의 문자가 바뀐다.

5. 시험 코드 읽는 순서: 모양을 배열 칸으로 통일한다

실기 문제는 문법을 한 개씩 묻지 않는다. 함수, 반복문, 조건문, 포인터, 문자열을 한 코드에 섞는다. 당황하지 않으려면 낯선 표기를 모두 몇 번째 칸을 읽거나 바꾸는가로 번역한다.

1단계

배열의 초기값을 칸으로 적는다. 문자열이라면 마지막에 '\0' 칸도 적는다.

2단계

포인터가 어디서 시작하는지 표시한다. p = arr + 1이면 두 번째 칸부터다.

3단계

*(p+i)와 p[i]를 배열 인덱스로 바꾸어 쓴다.

4단계

대입문을 만날 때마다 칸의 값을 고친다. 과거 값으로 계산하지 않는다.

5단계

반복문의 i 값과 조건 통과 여부를 한 행씩 기록한다.

6단계

마지막으로 %d, %c, %s가 무엇을 출력하는지 조립한다.

통합 예제 풀이

#include <stdio.h>

void edit(char *p) {
    for (int i = 0; p[i] != '\0'; i++) {
        if (i % 2 == 0) {
            p[i] = p[i] - 1;
        }
    }
}

int main() {
    char word[] = "DBDF";

    edit(word);
    printf("%s\n", word);
    return 0;
}
i p[i] 확인 i % 2 == 0 변경 현재 배열
0 'D' 참 'D' - 1 = 'C' C B D F \0
1 'B' 거짓 없음 C B D F \0
2 'D' 참 'D' - 1 = 'C' C B C F \0
3 'F' 거짓 없음 C B C F \0
4 '\0' 반복 종료 없음 "CBCF"
결과는 CBCF다. 이 문제는 문자열 문제처럼 보이지만 실제 작업은 char 배열의 0번과 2번 칸을 포인터 매개변수로 고친 배열 문제다.

6. 코딩 실습: 예상하고, 찍어 보고, 변형한다

각 실습은 실행 전 예상 → 종이 추적 → 실행 → 로그 추가 → 변형 순서를 지킨다. 온라인 컴파일러나 수업 환경에 코드를 입력하고, 예상 결과와 실제 결과가 다른 줄에 표시한다.

실습 A. 같은 칸을 네 가지 방식으로 읽기

#include <stdio.h>

int main() {
    int nums[4] = {11, 22, 33, 44};
    int *p = nums;

    printf("%d\n", nums[1]);
    printf("%d\n", *(nums + 1));
    printf("%d\n", p[1]);
    printf("%d\n", *(p + 1));

    return 0;
}
할 일: 네 줄 모두 같은 값이 나오는 이유를 nums[1]이라는 말로 설명한다. 그 다음 인덱스 1을 모두 3으로 바꾸어 다시 예측한다.

실습 B. 포인터로 배열 값 누적 변경

#include <stdio.h>

void addBefore(int *p, int size) {
    for (int i = 1; i < size; i++) {
        p[i] = p[i] + p[i - 1];
        printf("i=%d, current=%d\n", i, p[i]);
    }
}

int main() {
    int data[4] = {1, 2, 3, 4};
    addBefore(data, 4);

    for (int i = 0; i < 4; i++) {
        printf("%d ", data[i]);
    }
    return 0;
}
할 일: p[i - 1]은 이미 변경된 값임을 유의하여 최종 배열을 예측한다. 이후 초기 배열을 {2, 2, 2, 2}로 바꾸어 다시 표를 만든다.
첫 코드 결과 확인

중간 로그는 i=1, current=3, i=2, current=6, i=3, current=10이다.

최종 출력은 1 3 6 10이다. 직전 칸이 바뀐 뒤 다음 계산에 사용되는 누적형 문제다.

실습 C. 문자열의 끝 직접 찾기

#include <stdio.h>

int main() {
    char title[] = "C2026";
    int i = 0;

    while (title[i] != '\0') {
        printf("index=%d char=%c\n", i, title[i]);

할 일: title 메모리 칸을 '\0'까지 그린다. title[i]를 *(title + i)로 교체해도 출력이 같은지 확인한다.

실습 D. 특정 문자 개수 세기

#include <stdio.h>

int countChar(char *p, char target) {
    int count = 0;

    while (*p != '\0') {
        if (*p == target) {
            count++;
        }
        p++;
    }
    return count;
}

int main() {
    char text[] = "pointer and array";

    printf("%d\n", countChar(text, 'a'));
    return 0;
}
할 일: 결과를 예상한 뒤 target을 'r', 공백 문자 ' '로 바꾸어 본다. 포인터가 각 호출에서 처음부터 다시 출발하는 이유도 적는다.

실습 E. 문자열 안 숫자의 합

#include <stdio.h>

int digitSum(char *p) {
    int sum = 0;

    while (*p != '\0') {
        if (*p >= '0' && *p <= '9') {
            sum += *p - '0';
        }
        p++;
    }
    return sum;
}

int main() {
    char code[] = "A1B3C5";
    printf("%d\n", digitSum(code));
    return 0;
}
할 일: 왜 문자 '3'을 숫자 3으로 만들 때 '0'을 빼는지 설명한다. 문자열을 "2026-05-28"로 바꾸어 합을 예상한다.

7. 혼자 풀어 보는 실전 실기수준 문제

문제마다 먼저 출력 결과 또는 수정 코드를 종이에 쓴다. 해설은 자신의 추적표가 완성된 뒤 연다. 쉬운 문제에서 시작해 배열·포인터·문자열·함수를 한꺼번에 묻는 문제로 올라간다.

배열과 포인터 표기 변환

#include <stdio.h>

int main() {
    int a[4] = {4, 8, 12, 16};
    int *p = a;

    printf("%d %d %d\n", *p, *(a + 2), p[3]);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 4 12 16

*p는 a[0], *(a+2)는 a[2], p[3]은 a[3]이다.

포인터를 한 칸 이동한 뒤 읽기

#include <stdio.h>

int main() {
    int n[5] = {2, 4, 6, 8, 10};
    int *p = n + 1;

    printf("%d %d\n", *p, *(p + 2));
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 4 8

p = n + 1이므로 출발점은 n[1]이다. *(p+2)는 출발점에서 두 칸 뒤인 n[3]이다.

대입 순서를 놓치지 않기

#include <stdio.h>

int main() {
    int a[3] = {5, 10, 15};
    int *p = a;

    p[1] = p[0] + p[2];
    *(p + 2) = p[1] - p[0];

    printf("%d %d %d\n", a[0], a[1], a[2]);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 5 20 15

첫 대입으로 a[1]은 5 + 15 = 20이 된다. 다음 대입에서는 바뀐 p[1]을 사용하여 a[2] = 20 - 5 = 15가 된다.

문자열의 실제 칸 수

#include <stdio.h>

int main() {
    char s[] = "CAT";
    printf("%d %d %c\n", (int)sizeof(s), s[3], s[1]);
    return 0;
}

출력 결과를 쓰시오. 단, char는 1바이트이다.

정답과 해설

정답: 4 0 A

"CAT"는 'C','A','T','\0' 네 칸이다. s[3]은 종료 문자이며 정수로 출력하면 0이다.

문자열 포인터 반복

#include <stdio.h>

int main() {
    char s[] = "LEVEL";
    char *p = s;
    int count = 0;

    while (*p != '\0') {
        if (*p == 'L') {
            count++;
        }
        p++;
    }

    printf("%d\n", count);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 2

포인터는 L, E, V, E, L을 읽고 종료 문자에서 멈춘다. 첫 칸과 마지막 문자 칸이 'L'이다.

함수가 원본 문자열을 변경

#include <stdio.h>

void change(char *p) {
    p[0] = 'B';
    *(p + 2) = 'X';
}

int main() {
    char word[] = "CODE";
    change(word);
    printf("%s\n", word);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: BOXE

p[0]은 word[0]인 'C'를 'B'로 바꾼다. *(p+2)는 word[2]인 'D'를 'X'로 바꾼다.

반복문으로 문자 이동

#include <stdio.h>

void next(char *p) {
    for (int i = 0; p[i] != '\0'; i++) {
        if (p[i] >= 'A' && p[i] <= 'Z') {
            p[i]++;
        }
    }
}

int main() {
    char text[] = "ABC1";
    next(text);
    printf("%s\n", text);
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: BCD1

대문자인 첫 세 칸만 다음 문자로 바뀐다. '1'은 조건에 해당하지 않아 그대로다.

배열 합계를 포인터로 계산

#include <stdio.h>

int calc(int *p, int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) {
        if (i % 2 == 1) {
            sum += *(p + i);
        }
    }
    return sum;
}

int main() {
    int v[5] = {1, 3, 5, 7, 9};
    printf("%d\n", calc(v, 5));
    return 0;
}

출력 결과를 쓰시오.

정답과 해설

정답: 10

홀수 인덱스는 1, 3이다. 더해지는 값은 v[1] = 3, v[3] = 7이므로 10이다.

정답과 해설

수정: void first(int *p) 또는 void first(int p[])

배열 이름 data는 함수 호출에서 첫 칸 주소처럼 전달된다. 주소를 받아 인덱싱해야 하므로 포인터 또는 배열 매개변수로 선언한다. 출력은 99이다.

실기 종합: 문자와 인덱스가 함께 변한다

#include <stdio.h>

void solve(char *p) {
    int i = 0;
    while (p[i] != '\0') {
        if (i % 2 == 0) {
            p[i] = p[i] + 1;
        } else {
            p[i] = p[i] - 1;
        }
        i++;
    }
}

int main() {
    char s[] = "ABCD";
    solve(s);
    printf("%s\n", s);
    return 0;
}

출력 결과를 쓰고, 인덱스별 변경표를 작성하시오.

정답과 해설

정답: BADC

인덱스 원래 문자 조건 새 문자
0A짝수, +1B
1B홀수, -1A

도전: 숫자 문자 변환과 누적

#include <stdio.h>

int run(char *p) {
    int result = 0;

    while (*p != '\0') {
        if (*p >= '0' && *p <= '9') {
            result = result * 10 + (*p - '0');
        }
        p++;
    }
    return result;
}

int main() {
    char s[] = "A2B0C6";
    printf("%d\n", run(s));
    return 0;
}

출력 결과를 쓰고, result 변화표를 작성하시오.

정답과 해설

정답: 206

문자가 숫자일 때만 동작한다. '2'에서 0 * 10 + 2 = 2, '0'에서 2 * 10 + 0 = 20, '6'에서 20 * 10 + 6 = 206이다.

8. 마무리 정리와 자율 복습

오늘 반드시 남길 문장
  • 배열은 같은 자료형의 연속된 메모리 칸이다.
  • 배열 이름은 대부분의 식에서 첫 칸 주소처럼 쓰인다.
  • arr[i]는 *(arr+i)와 같은 칸이다.
  • 문자열은 '\0'로 끝나는 char 배열이다.
  • 함수에 배열을 넘기면 함수가 원본 칸을 수정할 수 있다.
혼동 금지
  • 배열과 포인터는 접근 방식이 비슷하지만 선언된 물건 자체는 다르다.
  • *p는 포인터의 값이 아니라, 가리킨 칸의 값이다.
  • p++는 다음 자료형 칸으로 이동한다.
  • %s는 '\0'까지 출력한다.
  • 대입 뒤에는 반드시 갱신된 배열값으로 다음 줄을 계산한다.

혼자 복습할 체크리스트

  • int arr[4] 메모리 그림을 그리고 *(arr+2)를 표시할 수 있다.
  • char s[] = "OK"가 몇 칸인지, 마지막 칸이 무엇인지 말할 수 있다.
  • while (*p != '\0')가 종료되는 순간을 설명할 수 있다.
  • 함수 매개변수 int *p 또는 char *p가 원본을 바꾸는 코드를 추적할 수 있다.
  • 연습문제 중 최소 3개를 값을 바꾸어 새 문제로 만들고 답을 다시 구했다.

변형 숙제

  1. 실습 B의 p[i] = p[i] + p[i - 1];를 p[i] = p[i] - p[i - 1];로 바꾸고 추적표를 작성한다.
  2. 실습 D에서 countChar를 고쳐 모음 a, e, i, o, u의 전체 개수를 세게 만든다.
  3. 연습문제 11번의 문자열을 "CODE"에서 "EXAM"으로 바꾸어 출력 결과를 다시 구한다.
  4. 직접 reverse(char *p) 함수를 작성해 "C-LANG"을 거꾸로 출력하거나 원본 배열 안에서 뒤집어 본다.
