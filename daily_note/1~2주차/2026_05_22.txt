==================================================
1. 반복문이 필요한 이유
==================================================

아래 코드를 보자.

```c
#include <stdio.h>

int main() {
    printf("Hello\n");
    printf("Hello\n");
    printf("Hello\n");
    printf("Hello\n");
    printf("Hello\n");

    return 0;
}
```

이 코드는 Hello를 5번 출력한다.
5번 정도는 직접 써도 된다.

하지만 100번 출력해야 한다면?
1000번 출력해야 한다면?

같은 코드를 계속 복사해서 쓰는 것은 좋지 않다.
이럴 때 반복문을 사용한다.

```c
#include <stdio.h>

int main() {
    int i;

    for (i = 0; i < 5; i++) {
        printf("Hello\n");
    }

    return 0;
}
```

이 코드는 Hello를 5번 출력한다.

반복문은 "같은 일을 여러 번 하라"는 명령이다.

==================================================
2. for문의 기본 구조
==================================================

for문의 기본 모양은 다음과 같다.

```c
for (초기식; 조건식; 증감식) {
    반복할 코드
}
```

예:

```c
for (i = 0; i < 5; i++) {
    printf("%d\n", i);
}
```

각 부분의 의미:

- 초기식: 반복을 시작하기 전에 한 번 실행한다.
- 조건식: 반복을 계속할지 판단한다.
- 실행문: 조건식이 참일 때 실행한다.
- 증감식: 실행문이 끝난 뒤 실행한다.

실행 순서:

```text
1. 초기식 실행
2. 조건식 검사
3. 조건식이 참이면 블록 실행
4. 증감식 실행
5. 다시 조건식 검사
6. 조건식이 거짓이면 반복 종료
```

반복문을 읽을 때는 항상 이 순서를 기억해야 한다.

==================================================
3. for문 첫 예제: i 값 출력
==================================================

예제 1:

```c
#include <stdio.h>

int main() {
    int i;

    for (i = 0; i < 5; i++) {
        printf("i = %d\n", i);
    }

    return 0;
}
```

출력:

```text
i = 0
i = 1
i = 2
i = 3
i = 4
```

왜 0부터 4까지 출력될까?

추적표:

| 반복 전/후 | i 값 | 조건 i < 5 | 실행 여부 |
|---|---:|---|---|
| 초기식 | 0 | 0 < 5 참 | 실행 |
| 1회 실행 후 i++ | 1 | 1 < 5 참 | 실행 |
| 2회 실행 후 i++ | 2 | 2 < 5 참 | 실행 |
| 3회 실행 후 i++ | 3 | 3 < 5 참 | 실행 |
| 4회 실행 후 i++ | 4 | 4 < 5 참 | 실행 |
| 5회 실행 후 i++ | 5 | 5 < 5 거짓 | 종료 |

중요:

```c
i < 5
```

이 조건은 i가 5일 때는 거짓이다.
그래서 5는 출력되지 않는다.

==================================================
4. 반복 횟수 세기
==================================================

아래 for문은 몇 번 반복될까?

```c
for (i = 0; i < 5; i++)
```

i 값은 0, 1, 2, 3, 4이다.
총 5번 반복된다.

아래 for문은?

```c
for (i = 1; i <= 5; i++)
```

i 값은 1, 2, 3, 4, 5이다.
총 5번 반복된다.

아래 for문은?

```c
for (i = 1; i < 5; i++)
```

i 값은 1, 2, 3, 4이다.
총 4번 반복된다.

정리:

| for문 | i 값 | 반복 횟수 |
|---|---|---:|
| i = 0; i < 5; i++ | 0,1,2,3,4 | 5 |
| i = 1; i <= 5; i++ | 1,2,3,4,5 | 5 |
| i = 1; i < 5; i++ | 1,2,3,4 | 4 |
| i = 5; i > 0; i-- | 5,4,3,2,1 | 5 |

정보처리기사 문제에서는 반복 횟수를 잘못 세면 답이 바로 틀린다.
반복문의 조건식을 반드시 손으로 확인한다.

==================================================
5. 실습 1: 반복 횟수 예상하기
==================================================

아래 코드의 출력 결과를 예상하라.

```c
#include <stdio.h>

int main() {
    int i;

    for (i = 2; i <= 6; i++) {
        printf("%d\n", i);
    }

    return 0;
}
```

i 값은 2, 3, 4, 5, 6이다.
출력:

```text
2
3
4
5
6
```

변형 실습:

1. i < 6으로 바꾸면?
2. i = 0으로 바꾸면?
3. i++를 i += 2로 바꾸면?

정답:

1. i < 6이면 2, 3, 4, 5 출력
2. i = 0, i <= 6이면 0부터 6까지 출력
3. i = 2, i <= 6, i += 2이면 2, 4, 6 출력

==================================================
6. printf 로그로 반복문 확인하기
==================================================

반복문은 눈으로만 보면 헷갈리기 쉽다.
공부할 때는 printf 로그를 적극적으로 찍어야 한다.

예제:

```c
#include <stdio.h>

int main() {
    int i;

    printf("반복 시작 전\n");

    for (i = 0; i < 3; i++) {
        printf("반복 안: i = %d\n", i);
    }

    printf("반복 끝난 후: i = %d\n", i);

    return 0;
}
```

출력:

```text
반복 시작 전
반복 안: i = 0
반복 안: i = 1
반복 안: i = 2
반복 끝난 후: i = 3
```

중요:
반복문 안에서는 i가 0, 1, 2일 때 실행된다.
반복이 끝난 뒤 i는 3이다.

왜냐하면 i가 2일 때 블록을 실행하고, 그 다음 i++가 실행되어 i가 3이 된다.
그 뒤 i < 3을 검사하면 거짓이므로 반복이 끝난다.

==================================================
7. 누적 계산: sum
==================================================

반복문에서 가장 자주 나오는 패턴은 누적 계산이다.

누적이란 기존 값에 계속 더해가는 것이다.

전날 배운 코드:

```c
sum = sum + i;
```

짧게 쓰면:

```c
sum += i;
```

예제 2:

```c
#include <stdio.h>

int main() {
    int i;
    int sum = 0;

    for (i = 1; i <= 5; i++) {
        sum += i;
    }

    printf("sum = %d\n", sum);

    return 0;
}
```

이 코드는 1부터 5까지 더한다.

추적표:

| 반복 | i | 반복 전 sum | 실행 코드 | 반복 후 sum |
|---|---:|---:|---|---:|
| 1회 | 1 | 0 | sum += 1 | 1 |
| 2회 | 2 | 1 | sum += 2 | 3 |
| 3회 | 3 | 3 | sum += 3 | 6 |
| 4회 | 4 | 6 | sum += 4 | 10 |
| 5회 | 5 | 10 | sum += 5 | 15 |

출력:

```text
sum = 15
```

==================================================
8. sum 초기값이 중요한 이유
==================================================

누적 변수는 반드시 초기값을 넣어야 한다.

좋은 코드:

```c
int sum = 0;
```

위험한 코드:

```c
int sum;
```

sum에 초기값을 넣지 않으면 이상한 값에서 시작할 수 있다.
그러면 더하기 결과도 이상해진다.

정보처리기사 문제에서는 초기값을 잘 봐야 한다.

예:

```c
int sum = 10;

for (i = 1; i <= 3; i++) {
    sum += i;
}
```

추적:

| i | sum |
|---:|---:|
| 시작 | 10 |
| 1 | 11 |
| 2 | 13 |
| 3 | 16 |

sum이 0에서 시작하는지, 10에서 시작하는지에 따라 답이 달라진다.

14. for문의 scope
==================================================

이제 scope를 반복문과 연결한다.

C언어에서는 변수의 scope가 선언 위치에 따라 달라진다.
반복문도 { } 블록을 만들기 때문에 scope가 생긴다.

예제:

```c
#include <stdio.h>

int main() {
    int i;

    for (i = 0; i < 3; i++) {
        int temp = i * 10;
        printf("temp = %d\n", temp);
    }

    return 0;
}
```

temp는 for문 블록 안에서 선언되었다.
따라서 for문 블록 안에서만 사용할 수 있다.

다음 코드는 오류이다.

```c
#include <stdio.h>

int main() {
    int i;

    for (i = 0; i < 3; i++) {
        int temp = i * 10;
    }

    printf("%d\n", temp);

    return 0;
}
```

이유:
temp는 for문 안에서 선언되었고, for문 밖에서는 사용할 수 없다.

수정:

```c
#include <stdio.h>

int main() {
    int i;
    int temp = 0;

    for (i = 0; i < 3; i++) {
        temp = i * 10;
    }

    printf("%d\n", temp);

    return 0;
}
```

출력:

```text
20
```

왜 20인가?
i가 0, 1, 2일 때 반복된다.
마지막 반복에서 temp = 2 * 10이므로 temp는 20이 된다.

==================================================
15. for문 안에서 선언한 i의 scope
==================================================

C99 이후 문법에서는 for문 안에서 i를 선언할 수 있다.

```c
for (int i = 0; i < 3; i++) {
    printf("%d\n", i);
}
```

이 경우 i는 for문 안에서만 사용할 수 있다.

아래 코드는 환경에 따라 컴파일 오류가 날 수 있다.

```c
#include <stdio.h>

int main() {
    for (int i = 0; i < 3; i++) {
        printf("%d\n", i);
    }

    printf("%d\n", i);

    return 0;
}
```

이유:
for문 안에서 선언한 i는 for문 밖에서 사용할 수 없다.

초보 수업에서는 헷갈리지 않게 아래처럼 쓰는 것을 권장한다.

```c
int i;

for (i = 0; i < 3; i++) {
    printf("%d\n", i);
}

printf("반복 후 i = %d\n", i);
```

이렇게 하면 i가 main 블록에서 선언되었기 때문에 for문 밖에서도 확인할 수 있다.

정보처리기사 문제에서는 둘 다 나올 수 있다.
따라서 i가 어디서 선언되었는지 반드시 확인해야 한다.

==================================================
16. 반복문 안쪽 변수는 매 반복마다 새로 만들어진다고 생각하기
==================================================

아래 코드를 보자.

```c
#include <stdio.h>

int main() {
    int i;

    for (i = 1; i <= 3; i++) {
        int temp = 0;
        temp += i;
        printf("%d\n", temp);
    }

    return 0;
}
```

출력:

```text
1
2
3
```

왜 1, 3, 6이 아닐까?

temp는 반복문 블록 안에서 선언된다.
각 반복마다 temp는 0으로 시작한다고 생각하면 이해하기 쉽다.

추적:

| i | temp 시작 | temp += i 후 |
|---:|---:|---:|
| 1 | 0 | 1 |
| 2 | 0 | 2 |
| 3 | 0 | 3 |

1, 3, 6을 만들고 싶다면 temp를 반복문 밖에서 선언해야 한다.

```c
#include <stdio.h>

int main() {
    int i;
    int temp = 0;

    for (i = 1; i <= 3; i++) {
        temp += i;
        printf("%d\n", temp);
    }

    return 0;
}
```

출력:

```text
1
3
6
```

핵심:

- 반복마다 새로 시작해야 하는 값은 반복문 안에서 선언해도 된다.
- 반복이 끝나도 누적되어야 하는 값은 반복문 밖에서 선언해야 한다.

17. 같은 이름 변수와 반복문 scope
==================================================

어제 배운 같은 이름 변수 문제가 반복문에서도 나온다.

```c
#include <stdio.h>

int main() {
    int i;
    int total = 100;

    for (i = 1; i <= 3; i++) {
        int total = 0;
        total += i;
        printf("안쪽 total = %d\n", total);
    }

    printf("바깥 total = %d\n", total);

    return 0;
}
```

출력:

```text
안쪽 total = 1
안쪽 total = 2
안쪽 total = 3
바깥 total = 100
```

왜 바깥 total은 106이 아닐까?

for문 안에서

```c
int total = 0;
```

을 다시 선언했기 때문이다.
이것은 바깥 total을 바꾸는 것이 아니라, 반복문 블록 안에서 사용할 새로운 total을 만든 것이다.

바깥 total을 누적하고 싶다면 int를 다시 쓰면 안 된다.

```c
#include <stdio.h>

int main() {
    int i;
    int total = 100;

    for (i = 1; i <= 3; i++) {
        total += i;
        printf("total = %d\n", total);
    }

    printf("마지막 total = %d\n", total);

    return 0;
}
```

출력:

```text
total = 101
total = 103
total = 106
마지막 total = 106
```

18. while문의 기본 구조
==================================================

while문은 조건식이 참인 동안 반복한다.

기본 구조:

```c
while (조건식) {
    반복할 코드
}
```

for문과 비슷하지만 초기식과 증감식이 괄호 안에 같이 들어 있지 않다.

예제:

```c
#include <stdio.h>

int main() {
    int i = 0;

    while (i < 5) {
        printf("%d\n", i);
        i++;
    }

    return 0;
}
```

출력:

```text
0
1
2
3
4
```

for문으로 쓰면:

```c
for (i = 0; i < 5; i++) {
    printf("%d\n", i);
}
```

for문과 while문은 서로 바꿔 쓸 수 있는 경우가 많다.

==================================================
19. for문과 while문의 차이
==================================================

for문은 반복 횟수가 비교적 분명할 때 많이 쓴다.

```c
for (i = 1; i <= 10; i++) {
}
```

while문은 조건이 만족되는 동안 반복할 때 많이 쓴다.

```c
while (number > 0) {
}
```

처음 공부할 때는 이렇게 기억하면 된다.

- for문: 몇 번 반복할지 보이는 반복
- while문: 조건이 참인 동안 계속하는 반복

하지만 정보처리기사에서는 둘 다 코드 추적 대상으로 나온다.
문법보다 중요한 것은 결국 변수값 변화이다.

==================================================
20. while문 추적
==================================================

예제 5:

```c
#include <stdio.h>

int main() {
    int n = 5;
    int count = 0;

    while (n > 0) {
        count++;
        n -= 2;
    }

    printf("n = %d\n", n);
    printf("count = %d\n", count);

    return 0;
}
```

추적:

| 반복 | 조건 n > 0 | n 시작 | count 변화 | n 변화 |
|---|---|---:|---:|---:|
| 1회 | 5 > 0 참 | 5 | 1 | 3 |
| 2회 | 3 > 0 참 | 3 | 2 | 1 |
| 3회 | 1 > 0 참 | 1 | 3 | -1 |
| 종료 검사 | -1 > 0 거짓 | -1 | 3 | -1 |

출력:

```text
n = -1
count = 3
```

중요:
while문은 조건을 먼저 검사한다.
조건이 처음부터 거짓이면 한 번도 실행되지 않는다.

==================================================
21. do while문의 기본 구조
==================================================

while문과 비슷하지만 조건을 검사하는 위치가 다른 반복문이 있다.
바로 do while문이다.

기본 구조:

```c
do {
    반복할 코드
} while (조건식);
```

do while문은 먼저 한 번 실행한 뒤, 마지막에 조건식을 검사한다.

실행 순서:

```text
1. do 블록 실행
2. 조건식 검사
3. 조건식이 참이면 다시 do 블록 실행
4. 조건식이 거짓이면 반복 종료
```

while문과 가장 큰 차이:

```text
while문은 조건을 먼저 검사한다.
do while문은 일단 한 번 실행하고 조건을 나중에 검사한다.
```

그래서 do while문은 조건이 처음부터 거짓이어도 최소 1번은 실행된다.

예제:

```c
#include <stdio.h>

int main() {
    int i = 0;

    do {
        printf("%d\n", i);
        i++;
    } while (i < 3);

    return 0;
}
```

출력:

```text
0
1
2
```

추적:

| 반복 | i 시작 | 출력 | i++ 후 | 조건 i < 3 |
|---|---:|---:|---:|---|
| 1회 | 0 | 0 | 1 | 참 |
| 2회 | 1 | 1 | 2 | 참 |
| 3회 | 2 | 2 | 3 | 거짓 |

주의:
do while문 끝에는 세미콜론이 붙는다.

```c
} while (i < 3);
```

이 마지막 세미콜론을 빠뜨리면 문법 오류가 난다.

==================================================
22. while문과 do while문의 차이
==================================================

두 코드를 비교하자.

--------------------------------------------------
코드 A: while문
--------------------------------------------------

```c
#include <stdio.h>

int main() {
    int n = 10;

    while (n < 5) {
        printf("while: %d\n", n);
        n++;
    }

    printf("end\n");

    return 0;
}
```

n은 10이다.
조건 n < 5는 처음부터 거짓이다.
따라서 while 블록은 한 번도 실행되지 않는다.

출력:

```text
end
```

--------------------------------------------------
코드 B: do while문
--------------------------------------------------

```c
#include <stdio.h>

int main() {
    int n = 10;

    do {
        printf("do while: %d\n", n);
        n++;
    } while (n < 5);

    printf("end\n");

    return 0;
}
```

do while문은 조건을 나중에 검사한다.
그래서 n < 5가 처음부터 거짓이어도 do 블록이 한 번은 실행된다.

출력:

```text
do while: 10
end
```

정리:

| 반복문 | 조건 검사 위치 | 처음 조건이 거짓이면? |
|---|---|---|
| while | 실행 전 | 0번 실행 |
| do while | 실행 후 | 1번 실행 |

==================================================
25. 무한 반복
==================================================

무한 반복은 반복문이 끝나지 않는 것이다.

아래 코드는 위험하다.

```c
#include <stdio.h>

int main() {
    int i = 0;

    while (i < 5) {
        printf("%d\n", i);
    }

    return 0;
}
```

i가 0에서 바뀌지 않는다.
조건 i < 5는 계속 참이다.
그래서 반복이 끝나지 않는다.

수정:

```c
while (i < 5) {
    printf("%d\n", i);
    i++;
}
```

반복문에서는 반복을 끝낼 수 있도록 조건에 사용되는 변수가 변해야 한다.

for문에서도 무한 반복이 생길 수 있다.

```c
for (i = 0; i < 5; i--) {
    printf("%d\n", i);
}
```

i는 0, -1, -2, -3처럼 계속 작아진다.
i < 5는 계속 참이다.
따라서 끝나지 않는다.

28. 반복문과 else if 결합
==================================================

반복문 안에서 else if도 사용할 수 있다.

```c
#include <stdio.h>

int main() {
```

| 6 | 참 | 검사 안 함 | 3 | 12 |

출력:

```text
12
```

i가 6일 때는 3의 배수이면서 2의 배수이다.

==================================================
30. 중첩 반복문 맛보기
==================================================

중첩 반복문은 반복문 안에 반복문이 들어가는 구조이다.
오늘은 깊게 들어가지 않고, 실행 흐름만 맛본다.

```c
#include <stdio.h>

int main() {
    int i;
    int j;

    for (i = 1; i <= 2; i++) {
        for (j = 1; j <= 3; j++) {
            printf("i=%d, j=%d\n", i, j);
        }
    }

    return 0;
}
```

출력:

```text
i=1, j=1
i=1, j=2
i=1, j=3
i=2, j=1
i=2, j=2
i=2, j=3
```

바깥 for문 i가 1일 때,
안쪽 for문 j가 1, 2, 3까지 모두 돈다.

그 다음 i가 2가 되고,
다시 안쪽 for문 j가 1, 2, 3까지 돈다.

총 반복 횟수:

```text
바깥 2번 * 안쪽 3번 = 6번
```

중첩 반복문은 3주차에 더 자세히 다룬다.
오늘은 "안쪽 반복문이 먼저 끝까지 돈다"만 기억한다.

```c
#include <stdio.h>
```

==================================================
32. 반복문에서 continue 맛보기
==================================================

continue는 이번 반복의 남은 부분을 건너뛰고 다음 반복으로 넘어간다.

```c
#include <stdio.h>

int main() {
    int i;

    for (i = 1; i <= 5; i++) {
        if (i == 3) {
            continue;
        }
        printf("%d\n", i);
    }

    return 0;
}
```

출력:

```text
1
2
4
5
```

i가 3일 때 continue가 실행된다.
그러면 아래 printf를 건너뛰고 다음 반복으로 넘어간다.

break와 continue 차이:

- break: 반복문 자체를 끝낸다.
- continue: 이번 반복만 건너뛰고 다음 반복으로 간다.

오늘의 중심은 for와 while이므로 break, continue는 맛보기만 한다.

49. 오늘의 핵심 정리
==================================================

1. 반복문은 같은 코드를 여러 번 실행한다.

```c
for (초기식; 조건식; 증감식) {
}
```

2. for문의 실행 순서:

```text
초기식 -> 조건식 -> 블록 실행 -> 증감식 -> 조건식 -> ...
```

3. 반복 횟수는 i 값 목록을 직접 적으면 안전하다.

```c
for (i = 0; i < 5; i++)
```

i는 0, 1, 2, 3, 4이다.

4. sum은 합계를 누적할 때 사용한다.

```c
sum += i;
```

5. count는 개수를 셀 때 사용한다.

```c
count++;
```

6. 반복문 안에 if문을 넣을 수 있다.

```c
for (i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        sum += i;
    }
}
```

7. while문은 조건이 참인 동안 반복한다.

```c
while (조건식) {
}
```

8. while문에서는 조건에 사용되는 변수를 반드시 변화시켜야 한다.

```c
while (i < 5) {
    i++;
}
```

9. do while문은 최소 한 번 실행된다.

```c
do {
    printf("%d\n", i);
    i++;
} while (i < 5);
```

while문은 조건을 먼저 검사하고, do while문은 실행 후 조건을 검사한다.

10. 반복문도 scope를 만든다.

```c
for (...) {
    int temp = 0;
}
```

temp는 반복문 밖에서 사용할 수 없다.

11. 반복문 밖에서도 누적값을 보고 싶다면 누적 변수는 반복문 밖에서 선언한다.

```c
int sum = 0;

for (...) {
    sum += i;
}

printf("%d\n", sum);
```

12. 반복문 안에서 같은 이름의 변수를 다시 선언하면 바깥 변수를 가린다.

```c
int sum = 0;

for (...) {
    int sum = 10;  // 새 변수
}
```

13. 공부할 때는 반복문 안에 로그를 찍는다.

```c
printf("i=%d, sum=%d\n", i, sum);
```

반복이 끝난 뒤 i는 10이다.
왜냐하면 i가 8일 때 실행한 뒤 i += 2가 실행되어 10이 되고,
10 <= 8은 거짓이므로 반복이 끝난다.

출력:

```text
10 20
```

숙제 2 정답
--------------------------------------------------

| i | 실행 | result |
|---:|---|---:|
| 시작 |  | 0 |
| 1 | 홀수, result -= 1 | -1 |
| 2 | 짝수, result += 2 | 1 |
| 3 | 홀수, result -= 3 | -2 |
| 4 | 짝수, result += 4 | 2 |
| 5 | 홀수, result -= 5 | -3 |

```text
-3
```

| 1 | 15 | 11 | 1 |
| 2 | 11 | 7 | 2 |

--------------------------------------------------
숙제 4 정답
--------------------------------------------------

오류가 난다.

last는 for문 블록 안에서 선언되었다.
for문 밖의 printf에서는 last를 사용할 수 없다.

수정:

```c
#include <stdio.h>

int main() {
    int i;
    int last = 0;

    for (i = 1; i <= 3; i++) {
        last = i * 10;
    }

    printf("%d\n", last);

    return 0;
}
```

출력:

```text
30
```

--------------------------------------------------
숙제 5 정답
--------------------------------------------------

for문 안의

```c
int total = 5;
```

은 새로운 안쪽 total을 만든다.
바깥 total은 바뀌지 않는다.

| i | 안쪽 total 시작 | total *= i 후 | 출력 |
|---:|---:|---:|---|
| 1 | 5 | 5 | 5 |
| 2 | 5 | 10 | 10 |
| 3 | 5 | 15 | 15 |

바깥 total은 계속 1이다.

출력:

```text
5
10
15
1
```

바깥 total을 바꾸려면 반복문 안의 int를 제거해야 한다.

```c
total *= i;
```

|---:|---:|---:|

반복:

| i | 조건 | 실행 | a | b |

| 2 | 짝수 | a += 2, b -= 1 | 3 | 8 |
| 3 | 홀수 | a += 1, b -= 3 | 4 | 5 |

반복이 끝난 뒤 i는 5이다.


--------------------------------------------------
숙제 7 정답
--------------------------------------------------

초기값:

| n | count |
|---:|---:|
| 12 | 0 |

do while문은 먼저 실행하고 나중에 조건을 검사한다.

| 반복 | n 시작 | n -= 5 후 | count | 조건 n > 0 |
|---|---:|---:|---:|---|
| 1회 | 12 | 7 | 1 | 참 |
| 2회 | 7 | 2 | 2 | 참 |
| 3회 | 2 | -3 | 3 | 거짓 |

출력:

```text
-3 3
```

n의 초기값을 -1로 바꾸면:

- while문은 조건을 먼저 검사하므로 n > 0이 처음부터 거짓이다. 따라서 0번 실행된다.
- do while문은 조건을 나중에 검사하므로 처음 조건이 거짓이어도 1번 실행된다.

==================================================
마무리 멘트
==================================================

오늘 배운 반복문은 정보처리기사 실기 C언어 문제의 핵심이다.
반복문 문제는 대부분 다음 세 가지를 묻는다.

1. 몇 번 반복되는가?
2. 반복할 때마다 변수값이 어떻게 바뀌는가?
3. 반복문 안에서 선언한 변수와 밖에서 선언한 변수를 구분할 수 있는가?

반복문을 볼 때는 바로 답을 쓰지 말고,
i 값 목록을 먼저 적는다.

그 다음 sum, count, result 같은 변수값을 한 줄씩 바꿔가며 표로 추적한다.

또한 scope를 계속 확인해야 한다.

- 반복문 밖에서 선언한 변수는 반복문 안에서 바꿀 수 있다.
- 반복문 안에서 선언한 변수는 반복문 밖에서 사용할 수 없다.
- 반복문 안에서 같은 이름으로 다시 선언하면 바깥 변수를 바꾸는 것이 아니다.

다음 수업에서는 배열 기초를 배운다.
배열은 반복문과 거의 항상 같이 사용된다.
그래서 오늘 배운 for문과 i 값 추적이 다음 수업의 가장 중요한 준비가 된다.
