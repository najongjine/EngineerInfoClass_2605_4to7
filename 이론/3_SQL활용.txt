p250
* SQL
 - DB 조작하는 명령어. 영어의 자연어를 닮음
 - DCL: 권한, DB 조작관련
   DDL: 테이블 생성, 수정, 삭제, 변경
   DML: 행(데이터) 조회, 추가, 수정, 삭제

* 관계대수
 - 데이터 가져오는 방식을 수학기호로 표현
 - 절차적
 - 일반집합 연산자 : 합집합(∪), 교집합(∩), 차집합(-), 교차곱(x)
    테이블과 테이블을 집합처럼 계산한다.
 - 순수관계 연산자 : SELECT(σ), PROJECT(π), JOIN(⋈), DIVISION(÷)
    테이블에서 원하는 행/열/관계를 뽑거나 연결한다.
* 관계해석
 - 원하는 데이터가 만족해야 하는 조건을 논리식으로 표현하는 
   **비절차적 질의 언어**이다

* 쿼리 최적화
 - 데이터가 10억개 있으면 검색하다가 DB 얼어버림
   많은 데이터를 빠르게 찾게하기위한 SQL을 만드는 기법
 - SQL 최적화라고도 함

p270
* 트랜젝션
 - 하나의 목적을 달성하기 위해 모여있는 SQL 들
  예: 목적은 송금, 사용하는 SQL들은 4개.
* 트랜잭션 특징 ACID
 - Atomicity(원자성): 모든 연사는 다 수행되거나 하나도 수행되지 말아야함
 - Consistency(일관성): 트랜잭션 작업 이후에, 데이터가 논리적이어야함
  예: 1000원 송금하면 나는 -1000되있고, 상대방은 +1000 되있어야함
 - Isolation(고립성): 트랜잭션 실행 도중 다른 트랜잭션의 영향을 받지 않아야함
 - Durability(지속성): 트랜잭션의 결과는 영구반영 되야함

* 트랜잭션 상태: 
 - 부분완료(partiality committed)
 - 완료(commit)
 - 실패(failed) 
 - 철회(aborted)
 - 취소(undo).   DBMS가 로그를 이용해 수행
 - 다시실행(redo)
 - rollback.   사용자/프로그램이 명령. ROLLBACK을 하면 내부적으로 UNDO가 수행됨
 - savepoint

* 트리거
 - DB 이벤트(예: 데이터 변경) 발생에 의해 자동으로 호출되는 SQL
 - 특정 이벤트에 따른 관련 작업을 자동으로 수행
  예: 누가 데이터 수정함 -> 수정일 자동 갱신

* 시스템 카탈로그
 - DB 안에 들어 있는 “DB 설명서” 또는 “DB 관리 대장”
   어떤 테이블이 있는지
   각 테이블에 어떤 컬럼이 있는지
   컬럼의 자료형은 무엇인지
   기본키는 무엇인지
   외래키는 무엇인지
   인덱스는 무엇인지
   사용자는 누구인지
   권한은 어떻게 되어 있는지
 - 메타데이터를 저장함
 - 데이터베이스에 저장된 모든 데이터 객체에 대한 정의나 명세 정보를 유지하는 시스템 테이블이다.
 - 이건 DB가 알아서 만드는것. 개발자나 사용자는 수정할수 없음


p278
* 병행제어
 - 여러 사용자들이 DB를 공동으로 사용하게 하는것
 - 여러사용자가 사용해도 DB 일관성을 유지 해야함
 - 일관성 유지를 안하면 송금하고, 자동이체하는 도중에 계산이 꼬이고, 
   잘못된 처리가 연쇠적으로 발생할수 있다.
* Locking
 - 트랜잭션이 갱신중인 데이터를 다른 트랜잭션이 접근하지 못하도록 잠그는 기술
  예: 내가 송금 하는 중에는 자동이체, 계좌조회 막음
 - 로크의 단위가 큰경우: 내가 백화점 쓸테니 다 나가.
      그러면 백화점 직원들은 편하겠죠. 하지만 백화점 사용자는 확 줄어들죠
      로크의 단위가 큰경우 병행성(공유도) 가 낮아지고 오버헤드가 감소한다.
 - 로크의 단위가 작은경우: 백화점의 직원 A씨만 고객한테 묶여있다
      그러면 손님들은 계속 자기 쇼핑하고 다른직원한테 물어보고, 일시키고
      로크의 단위가 작은경우 병행성(공유도)가 높아지고 오버헤드가 증가한다.
* 타임스탬프 기법
 - 트랜잭션이 순서대로(직렬) 처리될수 있도록 타임스탬프를 부여
* 낙관적 병행제어 (optimistic concurrency control)
 - 트랜잭션 수행동안은 어떠한 검사도 하지않고, 종료시에 일괄적으로 검사하는 기법
* 다중 버전 병행 제어(MVCC: Multi Version concurrency control)
 - 트랜잭션들의 타임스탬프를 비교하여 직렬 가능성이 보장되는 버전을 선택하는 기법


* 수평분할 수직분할
 - 테이블의 행(Row)을 기준으로 데이터를 나눕니다.
 - 테이블의 열(Column)을 기준으로 데이터를 나눕니
* 수평분할 기법
 -  범위 분할 (Range): 지정한 열의 값 범위(예: 날짜, 금액)를 기준으로 데이터를 나눕니다.
    해시 분할 (Hash): 해시 함수를 적용하여 데이터가 균등하게 분산되도록 나눕니다.
    목록 분할 (List): 특정 열에 명시된 코드나 값의 리스트를 기준으로 나눕니다.  
    라운드 로빈 (Round-robin): 해시나 특정 기준 없이, 일정한 크기 단위로 각 서버에 데이터를 순차적으로 할당합니다.
