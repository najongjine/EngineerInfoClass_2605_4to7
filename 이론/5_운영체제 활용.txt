p384
* 운영체제
 - 하드웨어와 사용자 사이에 위치하여 서로를 중재 해주는 놈
 - 하드웨어와 프로그램을 관리해줌
* window
 - GUI
 - 선점형 멀티태스킹, 병행처리
* UNIX
 - C언로 만듬, 병행처리
 - 유료
 - 다수의 사용자가 동시에 사용 가능
 - kernel: 핵심 시스템. 하드웨어 보호, 서비스, 프로세스, 메모리, 파일 관리
 - shell: 사용자 명령을 받아서 수행
 - utility: 편의성 도구들(메모장 같은거)
* LINUX
 - 리눅스 토발즈라는 괴물이 심심해서 혼자 무료 운영채제를 만듬
 - 유닉스 기반의 오픈소스 (UNIX에 있는 기능 다 되는데 무료임)
 - 마이크로 커널: 최소한의 핵심기능만 탑재한 커널
 - Monolithic kernel: 확장 불가능. 간단한 커널. 빠른속도
* Linux 명령어(shell 명령어)
 - ls (list): 디렉토리에 있는 목록들 보기
 - cp (copy): 복사
 - cat (concatenate): 파일 내용을 터미널에 출력하는 명령어
    예를 들어 memo.txt의 내용을 보고 싶다면
    cat memo.txt
 - mv (move): 파일이름 변경, 파일 이동
 - mkdir (make directory): 디렉토리 생성
 - chmod (change mod): 파일의 권한속성 변경
    예: 내가 파일 만듬. 누구는 보게하고 누는 안보게함
 - chown (change owner): 파일 소유자를 변경
    예: 내가 a.txt 만듬. 근데 나 퇴사할거임. 소유자 변경 해야함
 - mount: 원격 저장소 같은걸 내 컴터에 탑재시킴
    mount 하고나면 진짜 내 컴퓨터에 저장소 장착시킨거 아닌데, 꼭 장착된거 같음
 - grep: 문자열 패턴 검색

p390
* 프로세스
 - 프로그램(앱)이 메모리에 올라가서 진짜 실행될때 부르는 용어
 - 프로세스는 cpu에 할당되어 실행됨
 - 프로세스 정보는 PCB에 기록됨
* PCB(Process Control Block)
 - 프로세스 정보 기록하고 있는놈
    - 고유번호, 현재상태, 레지스터와 자원정보, 스케줄링 정보, 입출력 상태 정보
 - Time Slice에 의해 문맥교환(Context Switching)을 수행한다.
 - Time Slice가 작은경우 문맥교환 횟수, 오버헤드 증가
 - Time Slice가 큰은경우 문맥교환 횟수, 오버헤드 감소
* 프로세스 상태
 - 프로세스 상태는 생성, 준비, 실행, 대기, 종료 로 나뉜다
 - 우리가 쇼핑몰에 접속하면 
   생성 -> 준비 -> 실행
   사용중 디링~~~ 하면서 알람창 뜸
   실행에서 -> 대기   로 빠짐
   확인 버튼 누르면
   대기에서 -> 준비 -> 실행
 - 멀티태스킹: 사람이 느낄수 없는 아주 짧은 시간안에 (0.001~0.002초)
    수많은 프로그램들을 0.0001초씩 cpu를 할당해줘서 여러개를 자잘하게 아주 빠르게
    실행함
    그러면 사람 문에는 한꺼번에 움직이는거처럼 보임
 - Interrupt: 경고창 같은거. 경고창 뜨면 프로그램 클릭이 안됨. 이걸 Block 상태라고 함


 p392
 * 스레드   (요즘 프로그램은 복잡해서 성명 힘들고, 옛날 프로그램 기준)
 - 프로세스는 스레드 라는게 있음
    코드가 cpu에 의해서 하나씩 처리되는 장면을 시각화 하면
    꼭 실이 바늘구멍을 통과하는거와 비슷
    기본적으로 하나의 프로그램은 하나의 스레드로 통과됨
 - 만약, 여러 코드를 한꺼번에 실행 시키면
    시각화 해보면 여러개의 실이 바늘구멍을 통과하는거처럼 보임
    코드 여러줄이 한번에 실행되면 당연히 프로그램은 종료시간이 앞당겨짐
    이걸 멀티스레딩이라고 함
