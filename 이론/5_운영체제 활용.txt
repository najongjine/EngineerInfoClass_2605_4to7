* 프로세스 상태

   생성 -> 준비 -> 실행
   사용중 디링~~~ 하면서 알람창 뜸
   실행에서 -> 대기   로 빠짐
   확인 버튼 누르면
   대기에서 -> 준비 -> 실행

    실행함
    그러면 사람 문에는 한꺼번에 움직이는거처럼 보임

    이걸 멀티스레딩이라고 함

p393
* 비선점 스케줄링
 - 하나의 프로세스가 cpu 쓰고 있으면, 딴놈들은 다 끝날까지 기다려야함
* FIFO(First In First Out)   비선점
 - 프로세스가 도착한 순서대로 처리
* SJF(Shortest Job Jrst)   비선점
 - 실행 시간이 짧은 프로세스 순으로 처리 
 - 단점은 실행시간이 긴놈은 무한정 대기에 빠질수 있다
* HRN(Highest Response-ratio Next)   비선점
 - FIFO 와 SJF의 단점을 보안하여 개발된 방법
 - 우선순위 공식 = (대기시간)+(실행시간)/(실행시간)
 - 우선순위 공식이 큰 숫자가 1순위가 됨

* 선점 스케줄링
 - 딴 프로세스가 CPU 점유 뺏을수 있어요
* RR(Round Robin)
 - 특정 시간동안 프로세스가 cpu 사용하게 하고, 시간 지나면 다음놈한테 강제로 넘김
 - 실행 순서는 FIFO와 비슷
* SRT(Shortest Remaining Time)
 - 실행시간이 짧은 놈한테 넘김
 - SJF 랑 비슷
* MFQ
 - 프로세스를 여러 개의 대기열로 나누고, 
   실행 결과에 따라 우선순위를 올리거나 내리는 스케줄링 방식

   1. 잠금 확인

   3. 공유 자원 사용
   4. 사용이 끝나면 잠금 해제
* Mutex 
 - 상호배제를 위한 잠금 장치
 - 열쇠
* 상호배제의 조건
   1. 상호배제
   한 프로세스가 임계 구역을 사용 중이면 다른 프로세스는 들어갈 수 없습니다.

   2. 진행
   임계 구역이 비어 있으면 들어갈 프로세스를 결정해야 합니다.
   아무도 안 쓰는데 계속 기다리면 안 됩니다.

   3. 한정 대기
   특정 프로세스가 무한정 기다리면 안 됩니다.
   언젠가는 반드시 사용할 기회를 받아야 합니다.

* 교착상태
 - 다수의 프로세스가 같은 자원의 할당을 요구하며 무한정 기다리고 있는 상태
* 교착상태 필요 충분 조건
 - 상호 배제(mutual exclusion): 한 리소스는 한번에 한 프로세스만이 사용 가능
 - 점유와 대기(Hold and wait): 프로세스가 하나 이상의 리소스를 점유하고 있으면
      다른 프로세스가 가지고 있는 리소스를 기다리는 상태
 - 비선점(No preemption): 프로세스가 리소스를 자발적으로 반환할때까지 기다리는 상태
 - 환형대기(Circular wait): 각 프로세스가 원형처럼 다음 프로세스가 요구하는 
      자원을 가진 상태
* 교착상태 해결방안
 - 예방(prevention): 교착상태 필요충분 조건 4가지중 하나라도 만족이 안되면 교착상태 안일어난다
 - 회피(Avoidance): 은행원 알고리즘(Banker's Algorithm)이 있다
 - 회복(Recovery): 교착 상태가 발생한 프로세스중 프로세스 하나를 강제로 종료시켜
      자원을 뺃는다

 - 최초적합(First Fit): 데이터 배치가 가능한곳중 첫번째 공간에 배치

p399
* 내부 단편화  (박스를 받아서 채워 넣었는데, 박스에 빈공간이 남는다)
- 메모리를 나눠준 공간 안쪽에 남는 빈 공간입니다.
   예를 들어 프로그램이 7MB가 필요한데 운영체제가 10MB짜리 공간을 줬다면:
   [ 프로그램 7MB | 빈 공간 3MB ]
   남은 3MB는 다른 프로그램이 쓰기 어렵습니다.
* 외부 단편화
- 메모리 전체에 빈 공간은 충분하지만, 여러 곳에 흩어져 있어서 큰 프로그램을 넣지 못하는 현상입니다.
   [사용 4MB][빈 3MB][사용 5MB][빈 4MB][사용 2MB]
   빈 공간은 총 7MB입니다.
   하지만 6MB짜리 프로그램을 넣으려고 하면, 연속된 빈 공간이 최대 4MB라서 넣을 수 없습니다.
* 단편화 해결방법
 - 통합(coalescing): 서로 인접해 있는 공간 합침
 - 압축(compaction): 서로 떨어져 있는 공간까지 합침

** 페이지 교체 알고리즘
 - 페이지 프레임이 비어있을때(메모리에 필요한 데이터 없을때) 페이지를 적재한다(데이터를 메모리에 올린다)
 - 남는 공간이 있으면 그냥 올리면 되고, 남는 공간이 없으면 데이터를 쫒아내서 공간을 만든다
* OPT(Optimal replacement)    (상상의 기법)
 - 가장 오랫동안 사용하지 않을 페이지를 교체하는 기법
 - 가장 효율적이다
* FIFO(First In First Out)
 - 가장 먼저 적재된 페이지를 교체하는 방법
* LRU(Least Recently Used)
 - 최근에 가장 오랫동안 사용하지 않은 페이지를 교체
 - 각 페이지마다 stack 이라는 사용횟수 기록하는놈을 두어야함
* LFU(Least Frequently Used)
 - 사용빈도가 가장 적은 페이지 교체
 - 솔직히 LRU랑 같음
* NUR(Not Used Recently)
 - 최근에 가장 오랫동안 사용하지 않은 페이지를 교체
 - 솔직히 LRU랑 같음
* SCR(Second Chance Replacement)
 - FIFO의 단점을 보완
 - 자주 사용되는 페이지 교체 방지

* 페이지 크기 영향
 - 페이지 크기가 작은경우 페이지 단편화는 작고 페이지 입출력 횟수와 관리 부담은 많아진다
 - 페이지 크기가 큰경우 단편화는 많아지고, 페이지 입출력 횟수와 관리 부담은 줄어든다
