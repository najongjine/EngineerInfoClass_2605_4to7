CIA
* 기밀성(Confidentiality)
 - 인가된 사용자만 정보에 접근 가능함
* 무결성(Integrity)
 - 정보가 불법적으로 생성(위조), 변경(변조), 삭제 되지 않는 속성
* 가용성(Availabilty)
 - 인가된 사용자가 원하는때 사용 가능 해야함

* 인증성(Authentication)
 - 사용자 식별하기. 사용자 검증하기

* XSS(Cross Site Scripting)
 - 웹페이지에 악의적인 스크립트를 포함시켜 사용자측에서 실행되게끔 유도하는것
* 인젝션(injection)
 - 코드나 SQL을 주입하여 의도하지 않은 명령을 수행하게 만듬

p461
* 시스템 인증 구분
 - 알고있는것(지식기반 인증. Something You Know): 기억하고 있는것 으로 인증
  예: 비밀번호
 - 가지고 있는것(소유기반 인증. Something You Have) : 가지고 있는것 으로 인증
  예: 출입카드, 공인인증서
 - 스스로 모습 존재(존재기반 인증. Something You Are): 생체 정보로 인증
  예: 지문, 얼굴, 홍채
 - 행동하는것(행위기반. Something You Do): 서명, 움직임
 - 수행하는곳(위치기반 인증. Something You are): 행위자가 자리한곳의 물리적 위치를 통해 인증 수행
  예: 출석 하기, GPS

p462
* SSO(Single Sign On)
 - 단 한번의 로그인으로 한 기업의 모든 서비스 다 이용하기
  예: 네이버 로그인 하나로 네이버 웹툰, 카페, 쇼핑 할수 있다
* OAuth
 - 소셜 로그인
 - 어떤 쇼핑몰에 회원가입하지 않고 “Google로 로그인” 버튼을 눌러 로그인하는 것
* 디지털 서명
 - 네트워크에서 송신자의 신원을 증명하는 방법으로, 송신자가 자신의 비밀키로 암호화한 메시지를
  수신자가 송신자의 공개키로 해독하는 과정
 - 디지털 서명의 요구조건은 인증, 무결성, 부인방지, 유일성, 진위확인 용이성
*디지털 인증서
 - HTTPS, 공동인증서 에서 쓰임
 - HTTPS에서 “이 사이트가 진짜인지” 증명하는 전자 신분증
 - 공동인증서에서 “이 사람이 진짜 맞는지”

p464
* AAA (Authentication + Authorization + Accounting)
 - 인증 + 인가 + 계정관리/기록관리
 - Authentication: “너 누구야?”를 확인하는 단계
 - Authorization: “너 어디까지 할 수 있어?”를 정하는 단계
 - Accounting: “네가 무엇을 했는지 기록하는 단계”

p465
* DAC(임의적 접근통제. Discretionary Access Control)
 - 파일이나 자료의 주인이 “누가 접근할 수 있는지” 직접 정하는 방식
 - 소유자가 권한을 정한다
  | 사용자  | 권한         |
  | ---- | ---------- |
  | 나    | 읽기, 쓰기, 삭제 |
  | 친구 A | 읽기 가능      |
  | 친구 B | 접근 불가      |
  이렇게 문서 소유자인 내가 권한을 마음대로 설정할 수 있으면 DAC입니다.
* MAC(강제적 접근 통제. Mandatory Access Control)
 - 시스템이 정한 보안 등급대로 접근 허용
 - 군대
  | 사람  | 보안 등급 |
  | --- | ----- |
  | 김대리 | 일반    |
  | 박과장 | 비밀    |
  | 이부장 | 1급 비밀 |

  | 문서   | 보안 등급 |
  | ---- | ----- |
  | 회의록  | 일반    |
  | 작전문서 | 비밀    |
  | 핵심기밀 | 1급 비밀 |

  이때 일반 등급인 김대리는 1급 비밀 문서를 볼 수 없습니다.

* RBAC(역할 기반 접근 통제. Role-Based Access Control)
 - 역활
 - 역할에 따라 권한을 준다
 - | 역할  | 가능한 일              |
  | --- | ------------------ |
  | 사원  | 게시글 읽기             |
  | 팀장  | 게시글 읽기, 수정         |
  | 관리자 | 읽기, 수정, 삭제, 사용자 관리 |

  역할을 부여받으면 그것만 할수 있다
