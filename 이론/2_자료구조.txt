p124
* 동적계획법(Dynamic Programming)
 - 이미 계산된 문제의 해를 저장해 뒀다가 재사용하여 최적의 해를 구함
* 탐욕적 알고리즘(Greedy Algorithm)
 - 가장 최적의 선택? 해 나가며 문제를 해결
 - 하지만 최고의 선택이란 보장은 없음
* 재귀적 알고리즘(Recursive Algorithm)
 - 함수가 자기 자신을 또 호출하는 식으로 문제를 해결
* 근사 알고리즘(Approximation Algorithm)
 - 비교적 빠른 시간에 계산이 가능하도록 근사해법 수행
* 분할 정복(Dive and Conquer)
 - 커다란 문제를 조금씩 조금씩 쪼개면서 해결
* 퇴각 검색법(Backtracking)
 - 깊이우선탐색 DFS
 - 탐색하다가 탐색 실패하면 분기점이 있는 예전으로 돌아가기

p130
* 트리
 - 트리의 간선 개수 : N-1
 - 비순환 그래프
 - 노드의 차수(Degree): 자신의 자식 노드 개수
 - 트리의 차수: . 가장많은 자식노드가 어떻게되니. 최대 차수를 가지는 노드의 차수

* 트리순회 : 반바지

* 그래프 : 순환됨
 - 인접정점(Adjacent Vertext):
    A ----- B
    |
    C
    A와 B는 선으로 바로 연결됨 → 인접 정점
    A와 C도 선으로 바로 연결됨 → 인접 정점
    B와 C는 직접 선이 없음 → 인접 정점 아님
    
        B
        |
    A --- C --- D
    이 그래프에서 C의 인접 정점은?
    A, B, D
    C의 차수(Degree)는 3

p137
* O(1) : 1등
 ex: 1+99999+999999 (n 이 없음)

* O(N) : n 길이만큼 걸림
 ex: n + 9999 + 999n    100개다 100초. 1000개다 1000초
 for(i<n)

* O(NlogN) : n길이보다 살짝 더걸림
 ex: nlog(n) + 9999 + 999n    100개다 100초. 1000개다 1000초

* O(N**2) : N * N 만큼 걸린다
 ex: n**2 + 7876n + 990999
 for(i<n)
    for(j<n)

* O(N**3) : N * N * N 만큼 걸린다
    for(i<n)
        for(j<n)
            for(k<n)

* O(2**N) : 겁나 느리다

* O(N!): 프리징 걸릴정도로 느리다

p144
* 해시탐색
 -  O(1)
 - 데이터가 저장될 위치를 hash 함수라는걸 이용해서 바로 알아냄
 - 구현하기 어려운이유: 다른데이가 동일한 주소에 저장될려고 하면 충돌남
 - 충돌(collision): a,b 데이터가 같은 슬롯에 저장되려고 하는문제
 - 동의어(synonym): 충돌이 일어난 슬롯의 집합
 - 코드로 보면 어떻게 생겼어요? 
    Java에는 Map
    c언어는 struct
    python은 dictionary

* 해시함수 종류
 -제산법(Division): 키를 특정값으로 나눈 나머지값을 이용해서 저장위치 결정
 -폴딩법(Folding): 키를 합연산, XOR 연산해서 위치 결정
 -제곱법(Square): 키를 제곱한 결과로 위치 결정
 -기수변환법(Radix Conversion): 키값을 다른진법으로 변환
 -무작위법(Random): 난수를 이용
 -숫자분석법(Digital Analysis): 분포가 고른 부분

p146
* 삽입, 버블, 선택, 쉘 정렬은 메모리를 추가로 안써요
* 힙정렬, 이진병합, 퀵, 버킷 얘네들은 추가메모리를 써요
    그래서 추가메모리 쓰는 놈들은 시험문제로 잘 안나와요(퀵정렬 제외)

* 선택정렬: 왼쪽놈을 기준값으로 정하고, 제일 작은값을 찾고, 자리바꾸기 반복
* 버블정렬: 기존이 되는 데이터와 바로 오른쪽에 있는 데이터 비교후 자리 바꾸기
* 삽입정렬: 기준값이 2번째부터 시작함. 기준값과 지 왼쪽에 있느놈들을 비교후, 
    적절한 곳에 위치시킴
* 퀵정렬: pivot을 정함. pivot보다 작은놈은 왼쪽, 큰놈은 오른쪽.
    그러면 그룹이 나눠지는데, 나뉜 그룹에서 또 pivot 정하고 반복
