p414
* 단방향(simplex): 한 방향으로만 데이터 감
   예: TV, 라디오
* 반이중(Half-duplex): 데이터 전송, 수진 역활을 서로 바꿀수 있는 방식
   예: 무전기
* 전이중(full duplex): 데이터 전송 및 수신을 동시에 가능한 형태
   예: 전화기

* 동기화
 - 데이터 전송할때 송신측과 수신측의 송수신 작업 수행시간을 정확히 맞추는것
 - 동기식은 전송 효율과 속도가 높으며, 프레임 단위로 전송된다
 - 동기식은 전송비트에 동기화를 위한 제어 정보를 붙여서 프레임을 구성함
* 비동기식
 - 소량의 데이터를 저속으로 전송하며 문자 단위의 비트 블록을 전송함
* 동기식은 복잡하지만 고성능, 비동기식은 단순하고 싸지만 느림

* 동기식 제어문자
 - SYN: 동기화 하자고 알리는 문자
 - SOH: 데이터의 시작 부분(헤더)이라고 알리는 문자
 - Exx 이렇게 E 들어간건 데이터의 끝이라고 알리는 문자
 - ACK: 긍정 응답 문자
 - NAK: 부정 응답 문자

* 네트워크장비
* 허브(Hub)
 - 여러 대의 컴퓨터를 하나의 네트워크로 연결해 주는 장비
    예: 멀티탭
 - 어떤 컴퓨터가 데이터를 보내면, 그 데이터를 연결된 모든 컴퓨터에 뿌립니다
   예를 들어 A가 C에게 데이터를 보내도 B, C, D 모두에게 전송
 - 브로드캐스트 방식
 - 물리 계층 장비
* 리피터(Repeater)
 - 증폭기
 - 데이터를 멀리 보내면, 데이터 신호가 약해져요. 리피터가 증폭시켜서 신호를 다시 살아나게 해요
* Bridge
 - 두 개의 네트워크 구간을 연결하고, 필요한 데이터만 통과시키는 장비
 - 건물 A의 네트워크 ── 브리지 ── 건물 B의 네트워크
   구간 A                         구간 B
   PC 1 ─┐                       ┌─ PC 3
         ├─ 허브 ── 브리지 ── 허브 ┤
   PC 2 ─┘                       └─ PC 4
 - 네트워크 두 구간을 연결하면서, 
   MAC 주소를 보고 필요한 데이터만 반대편으로 보내는 장비
 - 2계층 장비
* 스위치(Switch)    얘도 어디까지나 멀티탭
 - 여러 컴퓨터를 연결하면서, 데이터가 필요한 컴퓨터에게만 보내주는 장비
 - L2~L4 수준까지의 스위치가 있고, L2 스위치는 데이터링크 계층, 2계층 장비, MAC 주소를 기억함
   주소 사무실, 학원같은데서 씀
   L3는 가정집, L4는 대기업
 - 여러 컴퓨터나 여러 네트워크 구간을 연결
* 라우터(Router)    장비
 - 서로 다른 네트워크를 연결하고, 데이터가 갈 길을 찾아주는 장비
 - 집 안 네트워크와 인터넷은 서로 다른 네트워크야.
   스마트폰 ─┐
   PC ──────┼─ 공유기/라우터 ─ 인터넷
   TV ──────┘
   라우터는 집 안에서 나온 데이터를 보고:
   “이 데이터는 인터넷으로 보내야겠네.”
   라고 판단해서 바깥 네트워크로 전달해.
 - IP 주소를 보고 판단
   목적지 IP 주소 확인
   → 어느 길로 보낼지 결정
 - L3 스위치: 회사 건물 안에서 부서 간 길을 연결 (라우팅 기능까지 추가해줌)
   공유기: 건물 안에서 외부 인터넷으로 나가는 출입구
* 게이트웨이(Gateway)      개념
 - 서로 다른 네트워크나 프로토콜을 연결하는 출입구 역할을 한다.

* 오류제어 방식 2가지 스타일
 - FEC(Forward Error Correction, 순방향, 전진): 자기가 오류 수정 가능
      해밍코드, 상승코드가 있다
 - BEC(Backward Error Correction, 역방향, 후진): 오류 수정은 못하고 체크만 가능
      페이티, ARQ, CRC ...
* 해밍코드(hamming)   FEC
 - 데이터의 오류검출 2비트, 직접수정 1비트
* 상승 코드   FEC
 - 여러 비트의 오류가 있더라도 여러비트 수정 가능
* Parity 검사   BEC
 - 비트수가 작수 혹은 홀수가 되도록 비트를 추가
  예: 001 -> 0011 로 짝수비트로 만듬
 - 2개의 오류가 동시에 발생하는 경우 검출이 불가능
* 블록합 검사    BEC
 - 패리티 검사 보완
    0011     가로로도 짝수 몇개인지 검사
    0011
    세로로도 짝수 몇개인지 검사
* CRC(Cyclic Redundancy Check,순한 중복 검사)   BEC
 - 오류가 한꺼번에 많이 발생하는 블록합 검사의 단점을 보완
 - *다항식* 코드를 사용하여 오류 검출

* ARQ(Automatic Repeat reQuest)
 - 오류가 발생할경우 자동으로 송신측에 데이터의 재전송을 요청하는 기술
* Stop and Wait ARQ
 - 송신측이 데이터 하나 보내고 수신측으로부터 응답신호(ACK,NAK)을 받을때까지 기다리는 방식
 - 프레임 1개 보내기 → ACK 응답 기다리기 → 다음 프레임 보내기
    정상케이스:
    송신자                    수신자
    │ ─── 데이터 1 ───────▶ │
    │ ◀──── ACK 1 ───────── │
    │ ─── 데이터 2 ───────▶ │
    │ ◀──── ACK 2 ───────── │
    │ ─── 데이터 3 ───────▶ │

    에러 케이스:
    송신자                    수신자
    │ ─── 데이터 1 ───────▶ │
    │                       │
    │     ACK가 안 옴        │
    │     시간 초과          │
    │ ─── 데이터 1 재전송 ─▶ │
    │ ◀──── ACK 1 ───────── │

* Go Back N ARQ
 - 한번에 여러 프레임을 보낸후 응답신호를 기다리고, 신호를 받으면 후속 데이터를 전송하는 방식
 - 송신측이 NAK을 받게되면 해당 프레임뿐 아니라 당시에 보냈던 모든 프레임을 재전송한다
* Selective repeat ARQ
 - go back n arq 방식을 보완
 - 오류가 난 부분만 재전송
 - 빠른 재전송이 가능하지만 수신측의 처리과정이 복잡함
* Adaptive ARQ
 - 전송 효율을 극대화 하기 위해 프레임의 길이를 동적으로 변경하여 전송
 - Selective repeat ARQ 업그레이드 버전


p420
* 다중화기(MUX:Multiplexer)
 - 여러 개의 저속 데이터 신호를 하나의 고속 전송 회선으로 합쳐 보내는 장치
  컴퓨터 A ─┐
  컴퓨터 B ─┼─▶ MUX ─▶ 하나의 고속 회선
  컴퓨터 C ─┘
  반대편에서는 역다중화기(DEMUX)가 다시 신호를 나눈다
  하나의 고속 회선 ─▶ DEMUX ─┬─▶ 컴퓨터 A
                            ├─▶ 컴퓨터 B
                            └─▶ 컴퓨터 C

* 주파수 분할 다중화(FDM, Frequency Division Multiplexing)
 - 주파수로 나눔
 - 전송 회선의 대역폭을 여러 개의 작은 주파수 대역으로 나눈다
 - TV, 라디오 등에서 사용하는 방식
 - 상호간섭, 잡음이 생길수 있음
* 시간분할 다중화(TDM, Time Division Multiplexing)
 - 시간으로 나눔
 - 전송 회선의 대여폭을 타임 슬롯으로 나누어 할당하는 방식
* 코드 분할 다중화(CDM, Code Division Multiplexing)
 - 코드로 나눔
 - 같은 시간과 같은 주파수를 사용하고, 서로 다른 코드로 사용자를 구분하는 다중화 방식
 -  A는 한국어로 말함
    B는 영어로 말함
    C는 일본어로 말함
    소리는 한 공간에 섞이지만, 듣는 사람이 특정 언어를 알고 있으면 그 사람의 말만 골라 들을 수 있습니다.
    
    예제:
    FDM
    A: 1번 주파수
    B: 2번 주파수
    C: 3번 주파수

    TDM
    1초: A
    2초: B
    3초: C

    CDM
    A, B, C가 같은 시간과 같은 주파수를 사용
    단, 각각 다른 코드 사용

* 파장 분할 다중화(WDM, Wavelength Division Multiplexing)
 - 광섬유에서 서로 다른 파장의 광신호를 하나의 회선으로 동시에 전송하는 다중화 방식
 - 빨간빛 → 사용자 A 데이터
  초록빛 → 사용자 B 데이터
  파란빛 → 사용자 C 데이터
  이 빛들을 한 광섬유에 합쳐서 동시에 전송
 

p421
* 집중화기(Concentrator)
 - 하나의 전송 회선을 하나의 단멀기가 독점하여 사용
 - 나머지들은 버퍼에서 대기
* CSMA(Carrier Sense Multiple Access)
 - 긍정 응답을 기다리지 않고 사용중인 채널에 일전한 규칙을 통하여 접근하는 방식
* CSMA/CD(Collision Detection)
 - CSMA 방식에 충돌 검출 기능과 재전송 기능 추가
 - 유선 LAN, Ethernet에서 사용
* CSMA/CA(collision avoidance)
 - 먼저 통신 채널이 사용 중인지 확인하고, 비어 있으면 잠시 기다린 뒤 전송하여 충돌을 최대한 피하는 방식
 - 무선, WIFI
 - Carrier Sense(반송파 감지) 기술이 회선 사용중인지 아닌지 알아 낸다고함
