p414
* 단방향(simplex): 한 방향으로만 데이터 감
   예: TV, 라디오
* 반이중(Half-duplex): 데이터 전송, 수진 역활을 서로 바꿀수 있는 방식
   예: 무전기
* 전이중(full duplex): 데이터 전송 및 수신을 동시에 가능한 형태
   예: 전화기

* 동기화
 - 데이터 전송할때 송신측과 수신측의 송수신 작업 수행시간을 정확히 맞추는것
 - 동기식은 전송 효율과 속도가 높으며, 프레임 단위로 전송된다
 - 동기식은 전송비트에 동기화를 위한 제어 정보를 붙여서 프레임을 구성함
* 비동기식
 - 소량의 데이터를 저속으로 전송하며 문자 단위의 비트 블록을 전송함
* 동기식은 복잡하지만 고성능, 비동기식은 단순하고 싸지만 느림

* 동기식 제어문자
 - SYN: 동기화 하자고 알리는 문자
 - SOH: 데이터의 시작 부분(헤더)이라고 알리는 문자
 - Exx 이렇게 E 들어간건 데이터의 끝이라고 알리는 문자
 - ACK: 긍정 응답 문자
 - NAK: 부정 응답 문자

* 네트워크장비
* 허브(Hub)
 - 여러 대의 컴퓨터를 하나의 네트워크로 연결해 주는 장비
    예: 멀티탭
 - 어떤 컴퓨터가 데이터를 보내면, 그 데이터를 연결된 모든 컴퓨터에 뿌립니다
   예를 들어 A가 C에게 데이터를 보내도 B, C, D 모두에게 전송
 - 브로드캐스트 방식
 - 물리 계층 장비
* 리피터(Repeater)
 - 증폭기
 - 데이터를 멀리 보내면, 데이터 신호가 약해져요. 리피터가 증폭시켜서 신호를 다시 살아나게 해요
* Bridge
 - 두 개의 네트워크 구간을 연결하고, 필요한 데이터만 통과시키는 장비
 - 건물 A의 네트워크 ── 브리지 ── 건물 B의 네트워크
   구간 A                         구간 B
   PC 1 ─┐                       ┌─ PC 3
         ├─ 허브 ── 브리지 ── 허브 ┤
   PC 2 ─┘                       └─ PC 4
 - 네트워크 두 구간을 연결하면서, 
   MAC 주소를 보고 필요한 데이터만 반대편으로 보내는 장비
 - 2계층 장비
* 스위치(Switch)    얘도 어디까지나 멀티탭
 - 여러 컴퓨터를 연결하면서, 데이터가 필요한 컴퓨터에게만 보내주는 장비
 - L2~L4 수준까지의 스위치가 있고, L2 스위치는 데이터링크 계층, 2계층 장비, MAC 주소를 기억함
   주소 사무실, 학원같은데서 씀
   L3는 가정집, L4는 대기업
 - 여러 컴퓨터나 여러 네트워크 구간을 연결
* 라우터(Router)    장비
 - 서로 다른 네트워크를 연결하고, 데이터가 갈 길을 찾아주는 장비
 - 집 안 네트워크와 인터넷은 서로 다른 네트워크야.
   스마트폰 ─┐
   PC ──────┼─ 공유기/라우터 ─ 인터넷
   TV ──────┘
   라우터는 집 안에서 나온 데이터를 보고:
   “이 데이터는 인터넷으로 보내야겠네.”
   라고 판단해서 바깥 네트워크로 전달해.
 - IP 주소를 보고 판단
   목적지 IP 주소 확인
   → 어느 길로 보낼지 결정
 - L3 스위치: 회사 건물 안에서 부서 간 길을 연결 (라우팅 기능까지 추가해줌)
   공유기: 건물 안에서 외부 인터넷으로 나가는 출입구
* 게이트웨이(Gateway)      개념
 - 서로 다른 네트워크나 프로토콜을 연결하는 출입구 역할을 한다.

* 오류제어 방식 2가지 스타일
 - FEC(Forward Error Correction, 순방향, 전진): 자기가 오류 수정 가능
      해밍코드, 상승코드가 있다
 - BEC(Backward Error Correction, 역방향, 후진): 오류 수정은 못하고 체크만 가능
      페이티, ARQ, CRC ...
* 해밍코드(hamming)   FEC
 - 데이터의 오류검출 2비트, 직접수정 1비트
* 상승 코드   FEC
 - 여러 비트의 오류가 있더라도 여러비트 수정 가능
* Parity 검사   BEC
 - 비트수가 작수 혹은 홀수가 되도록 비트를 추가
  예: 001 -> 0011 로 짝수비트로 만듬
 - 2개의 오류가 동시에 발생하는 경우 검출이 불가능
* 블록합 검사    BEC
 - 패리티 검사 보완
    0011     가로로도 짝수 몇개인지 검사
    0011
    세로로도 짝수 몇개인지 검사
* CRC(Cyclic Redundancy Check,순한 중복 검사)   BEC
 - 오류가 한꺼번에 많이 발생하는 블록합 검사의 단점을 보완
 - *다항식* 코드를 사용하여 오류 검출


p418
* ARQ(Automatic Repeat reQuest)
 - 오류가 발생할경우 자동으로 송신측에 데이터의 재전송을 요청하는 기술
* Stop and Wait ARQ
 - 송신측이 데이터 하나 보내고 수신측으로부터 응답신호(ACK,NAK)을 받을때까지 기다리는 방식
 - 프레임 1개 보내기 → ACK 응답 기다리기 → 다음 프레임 보내기
    정상케이스:
    송신자                    수신자
    │ ─── 데이터 1 ───────▶ │
    │ ◀──── ACK 1 ───────── │
    │ ─── 데이터 2 ───────▶ │
    │ ◀──── ACK 2 ───────── │
    │ ─── 데이터 3 ───────▶ │

    에러 케이스:
    송신자                    수신자
    │ ─── 데이터 1 ───────▶ │
    │                       │
    │     ACK가 안 옴        │
    │     시간 초과          │
    │ ─── 데이터 1 재전송 ─▶ │
    │ ◀──── ACK 1 ───────── │

* Go Back N ARQ
 - 한번에 여러 프레임을 보낸후 응답신호를 기다리고, 신호를 받으면 후속 데이터를 전송하는 방식
 - 송신측이 NAK을 받게되면 해당 프레임뿐 아니라 당시에 보냈던 모든 프레임을 재전송한다
* Selective repeat ARQ
 - go back n arq 방식을 보완
 - 오류가 난 부분만 재전송
 - 빠른 재전송이 가능하지만 수신측의 처리과정이 복잡함
* Adaptive ARQ
 - 전송 효율을 극대화 하기 위해 프레임의 길이를 동적으로 변경하여 전송
 - Selective repeat ARQ 업그레이드 버전


p420
* 다중화기(MUX:Multiplexer)
 - 여러 개의 저속 데이터 신호를 하나의 고속 전송 회선으로 합쳐 보내는 장치
  컴퓨터 A ─┐
  컴퓨터 B ─┼─▶ MUX ─▶ 하나의 고속 회선
  컴퓨터 C ─┘
  반대편에서는 역다중화기(DEMUX)가 다시 신호를 나눈다
  하나의 고속 회선 ─▶ DEMUX ─┬─▶ 컴퓨터 A
                            ├─▶ 컴퓨터 B
                            └─▶ 컴퓨터 C

* 주파수 분할 다중화(FDM, Frequency Division Multiplexing)
 - 주파수로 나눔
 - 전송 회선의 대역폭을 여러 개의 작은 주파수 대역으로 나눈다
 - TV, 라디오 등에서 사용하는 방식
 - 상호간섭, 잡음이 생길수 있음
* 시간분할 다중화(TDM, Time Division Multiplexing)
 - 시간으로 나눔
 - 전송 회선의 대여폭을 타임 슬롯으로 나누어 할당하는 방식
* 코드 분할 다중화(CDM, Code Division Multiplexing)
 - 코드로 나눔
 - 같은 시간과 같은 주파수를 사용하고, 서로 다른 코드로 사용자를 구분하는 다중화 방식
 -  A는 한국어로 말함
    B는 영어로 말함
    C는 일본어로 말함
    소리는 한 공간에 섞이지만, 듣는 사람이 특정 언어를 알고 있으면 그 사람의 말만 골라 들을 수 있습니다.
    
    예제:
    FDM
    A: 1번 주파수
    B: 2번 주파수
    C: 3번 주파수

    TDM
    1초: A
    2초: B
    3초: C

    CDM
    A, B, C가 같은 시간과 같은 주파수를 사용
    단, 각각 다른 코드 사용

* 파장 분할 다중화(WDM, Wavelength Division Multiplexing)
 - 광섬유에서 서로 다른 파장의 광신호를 하나의 회선으로 동시에 전송하는 다중화 방식
 - 빨간빛 → 사용자 A 데이터
  초록빛 → 사용자 B 데이터
  파란빛 → 사용자 C 데이터
  이 빛들을 한 광섬유에 합쳐서 동시에 전송

* 집중화기(Concentrator)
 - 하나의 전송 회선을 하나의 단멀기가 독점하여 사용
 - 나머지들은 버퍼에서 대기
* CSMA(Carrier Sense Multiple Access)
 - 긍정 응답을 기다리지 않고 사용중인 채널에 일전한 규칙을 통하여 접근하는 방식
* CSMA/CD(Collision Detection)
 - CSMA 방식에 충돌 검출 기능과 재전송 기능 추가
 - 유선 LAN, Ethernet에서 사용
* CSMA/CA(collision avoidance)
 - 먼저 통신 채널이 사용 중인지 확인하고, 비어 있으면 잠시 기다린 뒤 전송하여 충돌을 최대한 피하는 방식
 - 무선, WIFI
 - Carrier Sense(반송파 감지) 기술이 회선 사용중인지 아닌지 알아 낸다고함

* 교환 회선
 - 사용자 A ─ 교환망(회선+장비들) ─ 사용자 B
  필요할 때 연결
  여러 사용자가 회선을 공유
  전용 회선보다 저렴
  경로가 매번 달라질 수 있음
 - 회선교환 방식, 축적 교환 방식으로 나뉨
* 회선 교환 방식
 - 통신하기 전에 먼저 “전용 길”을 하나 잡아놓고, 통신이 끝날 때까지 그 길을 계속 쓰는 방식
    예: 전화
 - 단점은 회선 낭비가 생김
* 축적 교환 방식
 - 데이터를 중간 장비가 일단 저장했다가, 다음 목적지로 다시 보내는 방식
 - Store and Forward 방식
 - 회선을 효율적으로 씀
 - 단점은 지연시간이 많이 생긴다
 * 메시지 교환 방식 : 메시지 전체를 하나의 덩어리로 보냄
 * 패킷 교환 방식: 데이터를 작은 조각으로 나눠 보냄
  * 가상회선(virtual circuit)
   - 가상의(논리적) 전송 경로를 확보한 뒤 데이터 패킷을 전송
   - 패킷 전송 순서는 바뀌지 않으며 traffic이 많이 발생
  * 데이터그램(datagram)
   - 논리적인 경로의 확보 없이 자유롭게 보냄
   - 패킷 전송 순서가 바뀔수 있고 traffic이 적음

** 라우팅
 - 데이터 패킷을 전송하는데 있어 가장 빠르고 안정적인 경로를 설정하여 전송하는 기술
 - 동적 라우팅: 현재 기본셋팅. 통신망의 상태에 따라 동적인 경로 설정. 라우터끼리 정보 공유
 - 시험에는 내부 라우팅 프로토콜(IGP) 위주로 나온다
 - 외부 라우팅 : EGP, BGP
 * BGP
  - 회사 내부 라우터끼리 쓰는 프로토콜이 아니라, 
    인터넷 전체에서 큰 네트워크끼리 길을 정할 때 쓰는 라우팅 프로토콜
  - 인터넷 회사들끼리 “우리 쪽으로 오려면 이 길로 오세요”라고 경로를 교환하는 프로토콜
  - KT 네트워크
    SK 브로드밴드 네트워크
    LG U+ 네트워크
    네이버 네트워크
    구글 네트워크
    AWS 네트워크
    대학교 네트워크
    BGP는 이 큰 네트워크들 사이에서 사용됩니다.

* Distance Vector
 - 라우터가 목적지까지 가는 거리와 방향을 이웃 라우터에게 물어보면서 길을 찾는 방식
  1 라우터가 자기 라우팅 테이블을 가지고 있음
  2 일정 시간마다 이웃 라우터에게 테이블을 보냄
  3 이웃에게 받은 정보를 보고 더 짧은 경로가 있으면 수정함
  4 모든 라우터가 반복하다 보면 전체 경로가 안정됨
  * RIP
   - 라우터끼리 “목적지까지 몇 칸 걸리는지”를 주고받아서 길을 정하는 라우팅 프로토콜
   - “홉 수”로 길을 고른다
   - 최대 15홉,    16홉 = 도달 불가
   - Bellman-Ford
  * IGRP
   - Cisco가 만든 Distance Vector 라우팅 프로토콜 = RIP의 한계를 보완하려고 만든 것
   - RIP보다 더 큰 네트워크에서도 쓸 수 있고, 홉 수 말고 여러 기준을 고려하는 라우팅 프로토콜
  * EIGRP
   - Cisco가 IGRP를 개선해서 만든 라우팅 프로토콜이다. RIP보다 빠르고, IGRP보다 발전됨
* Link State Vector
 - 라우터가 전체 네트워크 지도를 만들어 놓고, 그 지도를 보고 최단 경로를 계산하는 방식
 - 네비게이션 비슷
  1 이웃 라우터 확인
  2 링크 상태 정보 생성
  3 전체 라우터에게 전파
  4 모든 라우터가 같은 지도 보유
  5 최단 경로 계산
  * OSPF
   - 라우터들이 전체 네트워크 지도를 만든 뒤, 가장 비용이 적은 경로를 계산하는 라우팅 프로토콜
   - Cost를 봄.   그 경로를 지나갈 때 드는 비용
   - Dijkstra

* ipv4
 - 8비트씩 4부분으로 구성되는 32비트 주소체계
 - 각 자리는 0~255, 즉 256개로 구성된다

* CIDR 
 - 192.168.0.0/24
  여기서 핵심은 뒤의 숫자다
  /24
  IP 32칸 중에서 앞 24칸은 네트워크 번호로 쓴다.
  남은 8칸은 컴퓨터 번호로 쓴다.

  네트워크 부분: 24비트
  호스트 부분: 8비트

* 서브넷팅
 - 큰 네트워크를 작은 네트워크 여러 개로 쪼개는 것
 - 192.168.10.25
  192.168.10   → 어느 회사, 어느 네트워크인가?
  25           → 그 네트워크 안의 몇 번 컴퓨터인가?
 - 서브넷 마스크는 “어디까지가 네트워크인지 알려주는 선”
  IP 주소:        192.168.10.25
  서브넷 마스크: 255.255.255.0
  CIDR /24
  255.255.255.0
  = 11111111.11111111.11111111.00000000
  = 앞에 1이 24개
  = /24
  앞 24비트는 네트워크 부분(나눠 줬다. 1은 못건드림)
  뒤 8비트는 호스트 부분(컴퓨터들 연결할수 있다. 0은 건드릴수 있음)
 - 공식
  2^호스트비트 수 - 2
  전체 0 → 네트워크 주소
  전체 1 → 브로드캐스트 주소
 - 예시
  192.168.10.0/24
  네트워크 비트 = 24
  호스트 비트 = 8
  2^8 - 2 = 256 - 2 = 254개 (256-8 - 2 해도 됨)
  [즉 이 네트워크에는 컴퓨터를 최대 254대까지 배치할 수 있습니다.]
  192.168.10.0 은 네트워크주소로 쓰이고
  192.168.10.255 는 브로드캐스트 주소로 쓰입니다
  그래서 실제 PC에게 줄 수 있는 주소는: 192.168.10.1 ~ 192.168.10.254
 - 예시2
  IP 주소가 192.168.1.75/26
  C class. 3 칸은 못건드림
  11111111.11111111.11111111.  11000000
  마지막칸에서 1의 갯수 = 2^2 = 4개의 회사한테 나눠줄수 있음
  마지막칸에서 0의 갯수 = 2^6 = 한 회사당 64개의 컴퓨터 연결할수 있음
  범위가 64씩 끊깁니다.
  192.168.1.0   ~ 192.168.1.63
  192.168.1.64  ~ 192.168.1.127
  192.168.1.128 ~ 192.168.1.191
  192.168.1.192 ~ 192.168.1.255

* VPN
 - 가상 사설망
 - 인터넷 위에 나만 쓰는 안전한 비밀 통로를 하나 만드는 기술 (이렇게 굴러가는거처럼 느껴진다)
 - 중국 유저 넷플 요청-> 정부(공용망은 정부가 설치함) -> 넷플 막힘
 - 중국유저 vpn회사 요청(암호화)-> 중국정부(별일 없어보이네) -> VPN회사 거침 -> 넷플 -> VPN 터널 -> 유저
 - VPN은 공중망인 인터넷을 이용하여 사설망처럼 안전하게 사용할 수 있도록 암호화 터널을 구성하는 가상 사설망 기술이다.
