5. 다음은  Java 코드에 대한 문제이다. 아래 코드를 확인하여 알맞는 출력값을 작성하시오.
public class Main {
 
  public static void main(String[] args) {
 
    int a=5,b=0;
 
    try{
      System.out.print(a/b);
    }catch(ArithmeticException e){ // 산술하다 에러
      System.out.print("출력1");
    }catch(ArrayIndexOutOfBoundsException e) { // 배열첨자 넘는 에러
      System.out.print("출력2");

      System.out.print("출력3");
    }catch(Exception e){ // 위에꺼 다 해당 안되면 어기로 빠짐
      System.out.print("출력4");
    }finally{ // 무조건 마지막에 실행됨
      System.out.print("출력5");
    }
  }
}

13. 다음은 Java 코드에 대한 문제이다. 아래 코드를 확인하여 알맞는 출력값을 작성하시오.
 public class Main {
    public static void main(String[] args) {
        new Child();
        System.out.println(Parent.total);
    }
}
class Parent {
    static int total = 0; // 18
    int v = 1;
 
    public Parent() {
        // total = 0 + 2
        total += (++v);
        // p.show로 가긴 함...
        show();    
    }
    // override 체크 함
    public void show() {
        total += total;
    }
}
class Child extends Parent {
    int v = 10; // 13
    // 여기먼저 진입, 그다음 부모생성자 체크
    public Child() {
        v += 2;
        // total = 6 + 12                v++
        total += v++;
        System.out.println(total);
        show();
    }
   
    @Override // 이걸로 해야함
    public void show() {
        // total +=  36
        total += total * 2;
        System.out.println(total);
    }
    
}

14. 아래는 디자인 패턴에 대한 설명이다. 알맞는 답을 보기에 골라 작성하시오. 
Adapter 
 - 서로 다른 인터페이스를 가진 클래스들을 연결해 사용 가능하게 한다.
 - 기존 클래스(Adaptee)를 원하는 인터페이스(Target)에 맞게 변환하는 어댑터(Adapter)를 만든다.
 - 기존 클래스를 감싸서(wrapper) 인터페이스를 변환해주는 역할을 한다. 
 - 이미 만들어진 코드의 모양이 지금 내가 쓰려는 모양과 안 맞을 때, 중간에 어댑터를 끼워서 맞춰 쓰는 패턴
 - 호환되지 않는 인터페이스를 변환하여 함께 사용할 수 있게 하는 패턴
 - 호환되지 않는 인터페이스
    변환
    재사용
    기존 클래스 수정 없이 사용
    Wrapper
