** 관계대수만 연습하세요. 관계해석은 이론만 챙겨하게요.
   관계해석은 통계학에서 쓰는거라 계산이 너무 힘들어서 실기에 관계해석 수식은 안나와요

## 학습 목표

이 학습지는 정보처리기사 실기에서 출제되는 데이터베이스 이론 문제 중 **관계대수**와 **관계해석**을 5시간 동안 학습할 수 있도록 구성한 자료이다.

학습 후 다음을 할 수 있어야 한다.

- 관계대수의 기본 연산자를 구분하고 식을 해석할 수 있다.
- 선택, 프로젝트, 조인, 디비전 연산을 실기 문제 수준으로 풀이할 수 있다.
- 관계해석의 의미와 표현 방식을 이해할 수 있다.
- 관계대수와 관계해석의 차이를 설명할 수 있다.
- 주어진 요구사항을 관계대수식 또는 관계해석식으로 표현할 수 있다.

# 1시간차: 관계 모델과 관계대수 개요

## 1. 관계형 데이터 모델 기본 용어

| 용어 | 의미 | 예시 |
|---|---|---|
| 릴레이션 | 행과 열로 구성된 테이블 | 학생, 수강, 과목 |
| 튜플 | 릴레이션의 행 | 학생 1명의 정보 |
| 속성 | 릴레이션의 열 | 학번, 이름, 학과 |
| 도메인 | 속성이 가질 수 있는 값의 범위 | 학년: 1, 2, 3, 4 |
| 차수(Degree) | 속성의 개수 | 학생(학번, 이름, 학과) = 차수 3 |
| 카디널리티 | 튜플의 개수 | 학생 데이터 5건 = 카디널리티 5 |

## 2. 관계대수란?

관계대수는 관계형 데이터베이스에서 원하는 데이터를 얻기 위해 사용하는 **절차적 질의 언어**이다.

즉, "무엇을 구할 것인가"뿐만 아니라 "어떤 연산 순서로 구할 것인가"를 표현한다.

예:

```text
σ학과='컴퓨터공학과'(학생)
```

의미:

```text
학생 릴레이션에서 학과가 '컴퓨터공학과'인 튜플을 선택한다.
```

## 3. 관계대수의 주요 연산자

| 연산자 | 이름 | 의미 |
|---|---|---|
| σ | 선택 | 조건을 만족하는 행을 추출 |
| π | 프로젝트 | 특정 열을 추출 |
| ∪ | 합집합 | 두 릴레이션의 튜플을 합침 |
| - | 차집합 | 한 릴레이션에서 다른 릴레이션의 튜플 제거 |
| ∩ | 교집합 | 두 릴레이션에 공통으로 있는 튜플 |
| × | 카티션 프로덕트 | 두 릴레이션의 모든 튜플 조합 |
| ⋈ | 조인 | 관련 있는 튜플을 결합 |
| ÷ | 디비전 | 모든 조건을 만족하는 튜플 검색 |

## 1. 선택 연산: σ

선택 연산은 조건을 만족하는 **행**을 추출한다.

형식:

```text
σ조건(릴레이션)
```

예:

```text
σ학과='컴퓨터공학과'(학생)
```

결과:

| 학번 | 이름 | 학과 | 학년 |
|---|---|---|---:|
| S01 | 김민수 | 컴퓨터공학과 | 2 |
| S03 | 박지훈 | 컴퓨터공학과 | 4 |
| S05 | 정유진 | 컴퓨터공학과 | 3 |

## 2. 프로젝트 연산: π

프로젝트 연산은 원하는 **열**을 추출한다. 중복 튜플은 제거된다.

형식:

```text
π속성목록(릴레이션)
```

예:

```text
π학과(학생)
```

결과:

| 학과 |
|---|
| 컴퓨터공학과 |
| 정보통신과 |
| 경영학과 |

## 3. 합집합: ∪

두 릴레이션의 튜플을 합친다. 합집합을 수행하려면 두 릴레이션은 **합병 가능**해야 한다.

합병 가능 조건:

- 두 릴레이션의 차수가 같아야 한다.
- 대응되는 속성의 도메인이 같아야 한다.

## 4. 차집합: -

첫 번째 릴레이션에는 존재하지만 두 번째 릴레이션에는 존재하지 않는 튜플을 구한다.

예:

```text
π학번(학생) - π학번(수강)
```

의미:

```text
수강 신청을 한 적이 없는 학생의 학번을 구한다.
```

결과:

| 학번 |
|---|
| S04 |

## 5. 교집합: ∩

두 릴레이션에 공통으로 존재하는 튜플을 구한다.

교집합은 다음처럼 차집합으로 표현할 수도 있다.

```text
R ∩ S = R - (R - S)
```

## 6. 카티션 프로덕트: ×

두 릴레이션의 모든 튜플 조합을 만든다.

예:

```text
학생 × 과목
```

학생이 5개 튜플, 과목이 3개 튜플이면 결과 튜플 수는 다음과 같다.

```text
5 × 3 = 15개
```

# 3시간차: 확장 관계대수 연산

## 1. 조인 연산: ⋈

조인은 두 릴레이션에서 관련 있는 튜플을 결합하는 연산이다.

대표적인 조인 종류:

| 조인 | 의미 |
|---|---|
| 세타 조인 | 비교 연산자를 사용하여 조건에 맞는 튜플 결합 |
| 동등 조인 | 조인 조건이 = 인 세타 조인 |
| 자연 조인 | 같은 이름의 속성을 기준으로 자동 결합하고 중복 속성 제거 |
| 외부 조인 | 조인되지 않는 튜플도 결과에 포함 |

## 2. 세타 조인

형식:

```text
R ⋈조건 S
```

예:

```text
학생 ⋈학생.학번=수강.학번 수강
```

의미:

```text
학생 릴레이션과 수강 릴레이션을 학번이 같은 조건으로 결합한다.
```

## 3. 자연 조인

자연 조인은 두 릴레이션에서 같은 이름의 속성을 기준으로 같은 값끼리 결합한다.

예:

```text
학생 ⋈ 수강
```

학생과 수강에는 공통 속성인 학번이 있으므로 학번을 기준으로 조인된다.

## 4. 조인을 활용한 질의

예: 데이터베이스 과목을 수강한 학생의 이름을 구하시오.

풀이:

1. 수강과 과목을 과목코드 기준으로 조인한다.
2. 과목명이 '데이터베이스'인 튜플을 선택한다.
3. 학생과 학번 기준으로 조인한다.
4. 이름을 프로젝트한다.

관계대수식:

```text
π이름(학생 ⋈ (σ과목명='데이터베이스'(수강 ⋈ 과목)))
```

결과:

| 이름 |
|---|
| 김민수 |
| 이서연 |
| 박지훈 |
| 정유진 |

## 5. 디비전 연산: ÷

디비전은 "모든"이라는 조건이 들어간 문제에서 자주 사용된다.

예:

```text
모든 과목을 수강한 학생의 학번을 구하시오.
```

관계대수식:

```text
π학번, 과목코드(수강) ÷ π과목코드(과목)
```

결과:

| 학번 |
|---|
| S05 |

이유:

```text
S05는 C01, C02, C03을 모두 수강했다.
```

## 6. 디비전 실기 풀이 감각

디비전 문제를 만나면 다음 단계를 확인한다.

1. "모든", "전체", "각각의"라는 표현이 있는지 확인한다.
2. 나누어지는 릴레이션은 보통 `대상 + 조건` 형태이다.
3. 나누는 릴레이션은 "모두 만족해야 하는 조건 집합"이다.
4. 결과는 조건 집합을 모두 만족한 대상이다.

예:

```text
모든 과목을 수강한 학생
```

대상:

```text
학번
```

조건 집합:

```text
전체 과목코드
```

디비전식:

```text
π학번, 과목코드(수강) ÷ π과목코드(과목)
```

# 4시간차: 관계해석

## 1. 관계해석이란?

관계해석은 원하는 데이터가 만족해야 하는 조건을 논리식으로 표현하는 **비절차적 질의 언어**이다.

관계대수와의 차이:

| 구분 | 관계대수 | 관계해석 |
|---|---|---|
| 성격 | 절차적 언어 | 비절차적 언어 |
| 초점 | 어떻게 구할 것인가 | 무엇을 구할 것인가 |
| 표현 | 연산자 기반 | 술어 논리 기반 |

## 2. 튜플 관계해석

튜플 변수를 사용하여 조건을 표현한다.

형식:

```text
{ t | P(t) }
```

의미:

```text
조건 P(t)를 만족하는 튜플 t의 집합
```

예:

```text
{ t | 학생(t) AND t.학과='컴퓨터공학과' }
```

의미:

```text
학생 릴레이션의 튜플 t 중 학과가 컴퓨터공학과인 튜플
```

## 3. 튜플 관계해석 예시

컴퓨터공학과 학생의 이름을 구하시오.

```text
{ t.이름 | 학생(t) AND t.학과='컴퓨터공학과' }
```

성적이 90점 이상인 학생의 학번을 구하시오.

```text
{ s.학번 | 수강(s) AND s.성적>=90 }
```

데이터베이스 과목을 수강한 학생의 이름을 구하시오.

```text
{ st.이름 |
  학생(st) AND 수강(sc) AND 과목(c)
  AND st.학번=sc.학번
  AND sc.과목코드=c.과목코드
  AND c.과목명='데이터베이스'
}
```

## 4. 도메인 관계해석

도메인 관계해석은 튜플 변수가 아니라 속성값을 나타내는 도메인 변수를 사용한다.

형식:

```text
{ <x1, x2, ...> | P(x1, x2, ...) }
```

예:

```text
{ <이름> | 학생(학번, 이름, 학과, 학년) AND 학과='컴퓨터공학과' }
```

## 5. 존재와 전체 조건

관계해석에서는 논리 기호가 자주 등장한다.

| 기호 | 의미 |
|---|---|
| ∃ | 존재한다 |
| ∀ | 모든 |
| AND | 그리고 |
| OR | 또는 |
| NOT | 아니다 |

예:

```text
{ st.이름 |
  학생(st) AND
  ∃sc(수강(sc) AND st.학번=sc.학번 AND sc.성적>=90)
}
```

의미:

```text
90점 이상인 성적이 하나 이상 존재하는 학생의 이름
```

## 6. "모든" 조건의 관계해석

모든 과목을 수강한 학생의 이름을 구하시오.

```text
{ st.이름 |
  학생(st) AND
  ∀c(과목(c) -> ∃sc(수강(sc)
    AND sc.학번=st.학번
    AND sc.과목코드=c.과목코드))
}
```

읽는 법:

```text
학생 st에 대해, 모든 과목 c마다 st가 그 과목을 수강한 수강 튜플 sc가 존재하면 st의 이름을 구한다.
```

# 마지막 정리

## 관계대수 핵심 암기

| 기호 | 읽는 법 | 실기 포인트 |
|---|---|---|
| σ | 시그마, 선택 | 조건에 맞는 행 |
| π | 파이, 프로젝트 | 필요한 열 |
| ⋈ | 조인 | 릴레이션 연결 |
| - | 차집합 | 없는 것, 제외 |
| ÷ | 디비전 | 모든 조건 |

## 관계해석 핵심 암기

```text
관계대수: 절차적, 연산자 중심, 어떻게 구할지 표현
관계해석: 비절차적, 논리식 중심, 무엇을 구할지 표현
```

## 실기 문제 풀이 순서

1. 출력해야 하는 속성을 확인한다.
2. 필요한 릴레이션을 찾는다.
3. 조건이 있으면 선택 연산을 생각한다.
4. 여러 릴레이션이 필요하면 조인을 생각한다.
5. "없는", "하지 않은"이 있으면 차집합 또는 NOT EXISTS를 생각한다.
6. "모든", "전체"가 있으면 디비전 또는 ∀ 조건을 생각한다.
7. 마지막에 프로젝트로 필요한 속성만 남긴다.

## 자주 틀리는 부분

| 실수 | 올바른 접근 |
|---|---|
| 프로젝트를 먼저 해서 조인에 필요한 속성을 없앰 | 조인 후 마지막에 프로젝트 |
| "모든" 문제를 단순 조인으로 풂 | 디비전 또는 ∀ 사용 |
| "참여하지 않은" 문제를 선택 조건으로만 풂 | 차집합 또는 NOT ∃ 사용 |
| 자연 조인에서 공통 속성명을 고려하지 않음 | 공통 속성 기준 자동 조인 확인 |
| 관계해석에서 연결 조건을 빠뜨림 | 사번, 코드 등 외래키 조건 반드시 작성 |

# 5시간 학습 운영 예시

## 1시간차

- 관계형 데이터 모델 용어 암기
- 관계대수와 관계해석 차이 이해
- 확인 문제 풀이

## 2시간차

- 선택, 프로젝트, 집합 연산 학습
- 학생, 수강, 과목 릴레이션으로 기본 문제 풀이

## 3시간차

- 조인과 디비전 집중 학습
- "모든"이 들어간 문제 반복 풀이

## 4시간차

- 튜플 관계해석과 도메인 관계해석 비교
- ∃, ∀, NOT 조건 연습

## 5시간차

- 실전 문제 15개 풀이
- 직접 풀어보기 문제 A-H 풀이
- 틀린 문제는 "선택, 프로젝트, 조인, 차집합, 디비전" 중 어느 유형인지 표시
